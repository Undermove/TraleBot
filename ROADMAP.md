# Roadmap — Бомбора (TraleBot mini-app)

> Живой документ. Обновляется Клодом в каждую сессию работы над проектом. Открывай когда хочешь увидеть «что сейчас в планах» и «что было решено раньше».

## Что это

Бомбора — Telegram мини-апп при TraleBot, собачка-гид для изучения грузинского. Модульное приложение поверх существующей базы данных словаря: алфавит, грамматика (глаголы движения), персональный словарь пользователя. Встраивается в бот через `SetChatMenuButton` (сейчас feature-flagged).

## Продуктовая философия

Железные правила, не подлежат пересмотру без отдельного обсуждения:

- **Без сердечек и жизней.** Никакого Duolingo-style gate-keeping, никаких «осталось 3 жизни, подожди 30 минут».
- **Pay-once, не подписка.** Один платёж — полный доступ. Модель как у Productify.
- **Пользователь выбирает путь.** Все модули и уроки открыты с первого дня. Знаешь алфавит — пропускаешь.
- **Проще, чем Duolingo.** Без стрик-шейминга, без тревожных уведомлений «ты пропустил день».
- **Без этнических акцентов в UI.** Никогда («маладэц» и подобное — запрещено).
- **Везде «мини-апп».** Не «мини-аб», не «мини аб».
- **Дизайн учит.** UI-элементы встроены в обучение (лоадер как грузинская буква, номера модулей как исторические грузинские числительные ა/ბ/გ, индекс в словаре на ქართული алфавите).

## Что сейчас в main

Версия: коммит `587618a` (PR #91, смёрджен 2026-04-11).

**Контент:**
- **Модуль «Алфавит»** — 33 буквы грузинского, 7 уроков по 4-5 букв. Теория с карточками + практика распознавания в обе стороны (буква→транслит, транслит→буква).
- **Модуль «Глаголы движения»** — 11 уроков. Теория + multi-choice квизы. Вопросы из `src/Trale/GeorgianVerbsOfMovement/questions*.json`, загружаются через `GeorgianQuestionsLoaderFactory`.
- **Модуль «Мой словарь»** — читает `VocabularyEntry` пользователя, генерирует RU→GE квиз. Auto-fallback на стартовый набор (20 слов) когда словарь пуст. Определение грузинской стороны по юникоду `U+10A0..U+10FF`.
- **Landing page** — публичная страница на `tralebot.com/` для визитов вне Telegram, с CTA на бот.

**Инфраструктура:**
- `MiniAppUserProgresses` — таблица XP/стрика/completed-lessons, привязана к `User.TelegramId`. Миграция `20260411133548_AddMiniAppUserProgress`.
- Telegram WebApp `initData` HMAC-валидация во всех защищённых эндпоинтах (`TelegramInitDataValidator`).
- Dockerfile с 3-стадийной сборкой: `node:20-alpine` собирает Vite-бандл, `dotnet/sdk:8.0` публикует, `dotnet/aspnet:8.0` рантайм.
- `BotConfiguration.MiniAppEnabled` — фича-флаг, сейчас `false` в проде. При `true` бот ставит chat menu button «🐶 Бомбора» всем юзерам.
- Deploy: `deploy/tralebot.yml` передаёт `BotConfiguration__MiniAppEnabled=false`.

**Стек:**
- Backend: .NET 8, EF Core, PostgreSQL, существующая TraleBot архитектура (Clean Architecture + CQRS).
- Frontend: React 18 + Vite + TypeScript + TailwindCSS, в `src/Trale/miniapp-src/`, билдится в `src/Trale/wwwroot/`.
- Маскот: inline SVG Бомбора (собачка) с moods: happy, cheer, sleep, think, guide.

**Текущее направление: Minankari** — грузинская перегородчатая эмаль минанкари (მინანქარი) + настольная игра.

Это направление разрабатывается в уже уместе сессии. В main сейчас более ранняя версия дизайна (sketchbook-era). Minankari — следующая миграция.

**Палитра:**
- `cream #FBF6EC` — фон
- `jewelInk #15100A` — текст и обводки
- `navy #1B5FB0` — primary accent
- `ruby #E01A3C` — secondary accent, пройденные метки
- `gold #F5B820` — tertiary, highlight, текущее состояние

- **Manrope** (variable, 400-800) для всего: заголовки, body, кнопки, метки

## Учебная программа A0 → B2

Полная программа из ~37 модулей. Все уроки открыты сразу (по философии продукта), но UI рекомендует путь в зависимости от уровня (beginner/intermediate). Онбординг при первом входе определяет стартовую точку.

### Фаза 1: Алфавит (A0) — 5 модулей, прогрессивное чтение

Ключевой принцип: НИКОГДА не показываем слово с буквой, которую ученик ещё не выучил. Русские буквы постепенно вытесняются грузинскими.

- **1.1 Гласные** (5 букв): ა ე ი ო უ → слоги, первые звуки
- **1.2 Лёгкие согласные** (5 букв): ბ გ დ მ ნ → დედა, მამა, ბაბა, ბინა, დიდი
- **1.3 Частые согласные** (5 букв): რ ს ტ ლ კ → სახლი, კარგი, მარილი
- **1.4 Тройки** (5 букв): ფ თ ქ პ ჰ — вводим понятие звонкий/придыхательный/абруптивный (ბ/ფ/პ, დ/თ/ტ, გ/ქ/კ). Минимальные пары: ფული vs პური
- **1.5 Оставшиеся** (11 букв, 2 урока): წ ჭ ძ ჯ ჟ შ ჩ ც ხ ღ ყ — сложные звуки, к концу читает любое грузинское слово

### Фаза 2: Выживание (A0→A1) — 3 модуля

- **2.1 Приветствия**: გამარჯობა, ნახვამდის, მადლობა, ბოდიში, როგორ ხარ
- **2.2 О себе**: მე მქვია..., საიდან ხარ?, მე ვარ რუსეთიდან
- **2.3 SOS-фразы**: არ მესმის, რამდენი ღირს?, სად არის...?

### Фаза 2.5: Числительные — 4 урока

- **Урок 1**: 1-10 (ერთი...ათი)
- **Урок 2**: 11-20 + двадцатеричная система (ოცი = 20)
- **Урок 3**: 21-100 (ოცდაერთი = 21, ორმოცი = 40, ასი = 100)
- **Урок 4**: 100+, порядковые (პირველი, მეორე), валюта (ლარი, თეთრი), дни недели, месяцы

### Фаза 3: Основы грамматики (A1) — 7 модулей

- **3.1 Местоимения + «быть»**: ვარ, ხარ, არის + отрицание
- **3.2 «Иметь»**: აქვს (вещь) vs ჰყავს (человек!)
- **3.3 Указательные**: ეს/ეგ/ის (тройная система: рядом/у тебя/далеко)
- **3.4 Базовые глаголы настоящего**: ვ- = я, -ს = он, паттерны спряжения
- **3.5 Падежи**: все 7 (სახელობითი, მოთხრობითი, მიცემითი, ნათესაობითი, მოქმედებითი, ვითარებითი, წოდებითი)
- **3.6 Послелоги**: -ში (в), -ზე (на), -თან (у), -დან (из), -მდე (до), -კენ (к), -ქვეშ (под), -თავზე (сверху), -უკან (за), -წინ (перед)
- **3.7 Прилагательные + сравнительные степени**: ლამაზი → უფრო ლამაზი → ყველაზე ლამაზი + неправильные (კარგი → უკეთესი → საუკეთესო) + наречия

### Фаза 4: Лексика по темам (A1-A2) — 6 модулей ✅ Готово

Кафе, Такси/город, Врач, Магазин, Знакомство, SOS

### Фаза 5: Глаголы в системе (A2→B1) — 8 модулей

- **5.1 Четыре класса глаголов**: переходные (Класс 1), непереходные (Класс 2), косвенные/чувства (Класс 3), стативные (Класс 4)
- **5.2 Версионные гласные**: -უ- (для кого-то), -ი- (для себя), -ე- (по месту)
- **5.3 Приставки направления**: და- შე- გა- ა- ჩა- წა- მო- მი- გადა- (каждый глагол учим С его приставкой)
- **5.4 Имперфект**: ვწერდი (я писал регулярно)
- **5.5 Аорист — переворот падежей**: NOM→ERG для подлежащего переходных. Начинаем с Класса 2 (без смены), потом Класс 1
- **5.6 Неправильные аористы**: ამბობს→თქვა, სვამს→დალია, ხედავს→დაინახა
- **5.7 Склонение местоимений**: ის→მან/მას/მისი, полные таблицы
- **5.8 Условные предложения**: реальные (თუ) и нереальные (რომ + conditional)

### Фаза 6: Продвинутая грамматика (B1→B2) — 6 модулей

- **6.1 Перфект (Серия III)**: подлежащее в дательном, эвиденциальное значение
- **6.2 Плюсквамперфект**: დამეწერა
- **6.3 Сослагательное наклонение**: все три серии
- **6.4 Причастия и отглагольные**: -ება/-ობა формы
- **6.5 Относительные предложения**: რომელიც (склоняется!)
- **6.6 Каузатив и потенциал**: აკეთებინებს, შეუძლია

### 31. Грузинские дни недели в приветствии Бомборы `[done]`

**Зачем:** Экран Dashboard открывается ежедневно. Добавление чипа с грузинским названием текущего дня недели превращает каждый визит в пассивный урок без дополнительных кликов.

**Обучающий элемент:** Чип показывает грузинское название дня (ორშაბათი, სამშაბათი...). По тапу открывается тултип с транслитерацией и переводом. ### 14. Магазин вкусняшек для Бомборы (XP-экономика) `[idea]`

Бомбору можно «кормить» вкусняшками, которые покупаются за XP. Это главный мотиватор для повторного прохождения уроков.

**Вкусняшки (от дешёвых к дорогим):**
- 🦴 Косточка — 10 XP (маленькая анимация: Бомбора грызёт)
- 🥩 Мяско — 30 XP (средняя анимация: Бомбора прыгает от радости)
- 🍖 Стейк — 60 XP (большая анимация: Бомбора танцует + конфетти)
- 🎂 Торт — 100 XP (эпическая анимация: фейерверк + Бомбора в восторге + звук)
- 👑 Королевский пир — 200 XP (ультра-анимация: золотой дождь + Бомбора в короне)

**Экономика XP (мотивирует проходить больше + на 100%):**
- Первое прохождение урока на 100% = 20 XP
- Первое прохождение на <100% = 0 XP (мотивация переделать)
- Повторное прохождение на 100% = 10 XP (мотивация повторять)
- Повторное прохождение на <100% = 5 XP
- Ежедневный бонус за первый урок дня = +5 XP

**Расчёт:**
- 1 урок на 100% = 20 XP → хватает на 2 косточки
- 3 урока в день = 60 XP → хватает на стейк
- 5 уроков в день = 100 XP → хватает на торт
- Чтобы купить «Королевский пир» (200 XP) нужно ~10 уроков или 2-3 дня активной игры

**UI:**
- На дашборде рядом с Бомборой кнопка «Покормить» (или тап на Бомбору)
- Открывается попап с полкой вкусняшек
- У каждой вкусняшки цена в XP
- Недоступные (не хватает XP) затемнены
- При покупке: XP списывается, проигрывается анимация (чем дороже — тем ярче)
- После анимации — сытость Бомборы растёт, настроение меняется
- Rewarding-анимации должны быть такими, чтобы хотелось показать другу

**Хранение:**
- `MiniAppUserProgress.TotalTreatsGiven` — сколько раз покормили (бэкенд)
- `MiniAppUserProgress.XpSpent` — сколько XP потрачено на вкусняшки
- Доступные XP = `Xp - XpSpent`

Экспериментально. Распознавание грузинской речи для упражнений «say after me» и оценка произношения.

Отдельный экран для сложных звуков: ყ, ღ, წ, ჭ, ძ, ც. Объяснение артикуляции, контраст пар (პ vs ფ, ქ vs კ), слушай-повторяй.

### Монетизация — Telegram Stars ⭐

Оплата идёт через **Telegram Stars** (валюта `XTR`), а не через внешний payment provider. Это нативный способ для мини-аппов и ботов: у пользователя уже есть звёзды в его Telegram-кошельке, ему не надо вводить карту.

**Модель:**
- Free: Алфавит + Глаголы движения + Мой словарь (до 50 слов)
- Pro: все курсы + тематическая лексика + AI-тутор + TTS + Мой словарь без лимита

**Техника:**
- Один разовый invoice через `sendInvoice` с `currency: "XTR"` (не RUB/USD). Amount в звёздах (TBD, ориентир 100-300 звёзд за Pro).
- `BotConfiguration.PaymentProviderToken` для Stars НЕ нужен — это отдельный нативный поток Telegram.
- Новый эндпоинт или расширение существующего `PayCommand` под режим Stars.
- После успешной оплаты — хендлер `pre_checkout_query` → `successful_payment` → ставим флаг `IsPaidPro` (или даты покупки) на `User` в БД.
- Мини-апп читает флаг через `/api/miniapp/me` и разблокирует платные модули.
- Refund флоу через Telegram refund API (обязательно для Stars — ТГ требует кнопку возврата в течение 21 дня по их правилам).

**Существующий PaymentProviderToken остаётся** для обычной платёжки в основном боте (если ещё используется), но для мини-аппа Pro-доступа — только Stars.

**Open question:** сколько звёзд за Pro? 100, 150, 299? Звёзды ~ $0.013 каждая, то есть 299 ≈ $4. Это надо определить на базе целевого ценника в долларах.

## Агентный пайплайн

Разработка ведётся четырьмя автономными агентами. Промпты в `.claude/agents/`.

### Роли

| Агент | Частота | Вход | Выход |
|-------|---------|------|-------|
| **Product** | ежедневно (если `[idea]` < 5) | ROADMAP, issues, PR | новые `[idea]` задачи или приоритизация |
| **Designer** | ежедневно | задачи `[idea]` | спеки в `design-specs/`, статус → `[designed]` |
| **Developer** | ежедневно | задачи `[designed]` | PR с кодом, статус → `[review]` |
| **Tech Lead** | ежедневно | PR на ревью | approve/request changes, статус → `[done]` |

### Статусы задач

- `[idea]` — задача сформулирована, ждёт дизайн-спеку
- `[designing]` — Designer работает над спекой
- `[designed]` — спека готова в `design-specs/`, можно реализовывать
- `[dev]` — Developer реализует
- `[review]` — PR на ревью у Tech Lead
- `[done]` — PR одобрен, ждёт мёрж от владельца
- `[blocked]` — заблокирована другими задачами

### Правила

- **Design-Driven**: ничего не реализуется без спеки. Developer берёт только `[designed]` задачи.
- **Мёрж только вручную**: агенты не мёржат PR, это делает владелец проекта.
- **Один агент — одна задача за сессию**: не пытаться сделать всё сразу.
- **Обучающий элемент обязателен**: каждая фича должна содержать элемент, который учит грузинскому через UI.

## Как работать с этим файлом

- Обновляй при каждой сессии, где принимаются продуктовые решения
- Двигай пункты из «Идеи» → «Среднесрочное» → «Ближайшая итерация» по мере приоритизации
- Закрытые пункты уходят в «Лог решений» с датой
- Не бойся удалять неактуальные планы — roadmap это не архив, это план на сейчас
- Статусы `[idea]`/`[designing]`/`[designed]`/`[dev]`/`[review]`/`[done]` управляются агентами автоматически
