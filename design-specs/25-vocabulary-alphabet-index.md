## Обучающий элемент

**Что учит:** весь грузинский алфавит в порядке — ა ბ გ დ ე ვ ზ თ ი კ ლ მ ნ ო პ ჟ რ ს ტ უ ფ ქ ღ ყ შ ჩ ც ძ წ ჭ ხ ჯ ჰ — отображается как навигационная полоса. Пользователь ищет слова, а параллельно скользит взглядом по грузинским буквам в алфавитном порядке. Через неделю использования алфавитный порядок запоминается без специального заучивания.

**Паттерн:** привычный элемент (алфавитный указатель, как в бумажном словаре) → грузинский контекст (буквы в порядке ანბანი) → момент узнавания при прохождении Алфавитного модуля.

**Reveal-момент:** в модуле «Алфавит», урок 1.4 (тройки: ფ-თ-ქ-პ-ჰ), пользователь видит букву ქ и вспоминает: «я её уже видел в индексе словаря, она шла после ფ». Буква перестаёт быть абстрактной — у неё уже есть место в алфавите в памяти пользователя.

**Дополнительный обучающий момент:** состояние «буква без слов» (задизейблена, серая) ненавязчиво сообщает пользователю, что грузинские слова начинаются с этих букв. Мотивирует переводить больше слов, чтобы «разжечь» новые буквы.

**Буквы в порядке ანბანი:**

(33 буквы; «все» — не отдельная кнопка, а состояние «нет выбранной буквы»)

**Layout контейнера:**
```

padding: px-5 py-2 (горизонтальный padding = отступ контента страницы)

```

**Каждая кнопка-буква:**
```
min-w-[36px] h-[36px]
border-[1.5px] border-jewelInk rounded-lg
font-geo font-bold text-[15px]
tap target: 36×36px (≥ 44px с gap — принято, т.к. это secondary nav)
```

**Состояния кнопки:**

| Состояние | BG | Текст | Border | Shadow |
|-----------|-----|-------|--------|--------|
| Default (есть слова) | `cream-tile` | `jewelInk` | `jewelInk` | `2px 2px 0 #15100A` |
| Active (выбрана) | `navy` | `cream` | `jewelInk` | `none` (притаплена: 2px offset → 0) |
| Disabled (нет слов) | `cream-tile` | `jewelInk/25` | `jewelInk/20` | `none` |

**Активная буква при прокрутке:**
- При выборе буквы — автоматически скроллить её в видимую область (`.scrollIntoView({ inline: 'center', behavior: 'smooth' }`)
- При первом рендере — если есть активная буква, прокрутить к ней

**Сброс фильтра:**
- Повторный тап на активную букву снимает фильтр (activeLetter → null, показываются все слова)
- Семантически: «тапни ещё раз, чтобы убрать фильтр»

### Экран: Словарь — состояние с активным фильтром буквы

**Layout списка слов:**
```
[ ── ა ──────────────────────────── 3 слова ── ]
  [ jewel-tile: ათი — десять ]
  [ jewel-tile: ანბანი — алфавит ]
  [ jewel-tile: ახლა — сейчас ]
```

Над списком слов — тонкая разделительная строка с Georgian-буквой и счётчиком слов:
- Левый элемент: активная буква, `font-geo text-[20px] font-bold text-jewelInk` — крупно
- `flex-1` разделитель: `h-px bg-jewelInk/15`
- Правый элемент: `N слов`, `mn-eyebrow`
- Высота строки: ~28px

Если фильтр буквы активен + фильтр чипа `новые` — показываются слова, начинающиеся на эту букву И помеченные как новые. Комбинация работает аддитивно.

**Пустое состояние (буква есть, слов нет при комбинированном фильтре):**
```
[ ── ა ─────────────────────── ]
  [ нет слов по этому фильтру  ]
  [ (серый hint-текст центром) ]
```
Если же буква задизейблена — пользователь не может её нажать, пустое состояние не достигается.

### Экран: Словарь — loading, auth-required, error, starter

Алфавитный индекс **не показывается** в этих состояниях. Он появляется только в `phase === 'ready'` и `!isStarterMode`.

В starter-режиме (словарь пуст, показываются стартовые слова):
- Индекс не показывается — стартовая колода Бомборы не требует алфавитной навигации
- Через несколько переведённых слов пользователь переходит в non-starter режим, индекс появляется

## Логика фильтрации

**Алфавитный фильтр применяется к Georgian-стороне слова:**
```typescript
// Используется существующая функция sides(item) из VocabularyList.tsx
function matchesAlpha(item: VocabularyItem, letter: string): boolean {
  const { georgian } = sides(item)
  return georgian.startsWith(letter)
}
```

**Комбинирование с существующим фильтром:**
```typescript
const filtered = items.filter((item) => {
  // 1. Существующий mastery-фильтр (все/новые/сложные/изучено)
  if (filter === 'new' && ...) return false
  if (filter === 'weak' && ...) return false
  if (filter === 'mastered' && ...) return false
  // 2. Существующий текстовый поиск
  if (search && !matchesSearch(item, search)) return false
  // 3. Новый: алфавитный фильтр
  if (activeLetter && !matchesAlpha(item, activeLetter)) return false
  return true
})
```

**Disabled-буквы (нет слов по этой букве в полном словаре — без учёта mastery-фильтра):**
```typescript
const disabledLetters = new Set(
  GEORGIAN_ALPHABET.filter(letter =>
    !items.some(item => sides(item).georgian.startsWith(letter))
  )
)
```
Disabled считается по **всем** словам, не только отфильтрованным — буква не становится серой просто потому что включён фильтр «новые».

## Адаптивность (375px)

**На минимальной ширине 375px:**
- Видимых букв в строке без скролла: ~8-9 (при `min-w-[36px]` + `gap-6px` = 42px на букву, 375−40px отступов = 335px ÷ 42 ≈ 8 букв)
- Остальные 25 букв доступны горизонтальным скроллом
- Горизонтальный скролл: `overflow-x: auto`, `-webkit-overflow-scrolling: touch`, `scroll-snap-type: none` (не нужен snap)
- Полоса прокрутки скрыта (`::-webkit-scrollbar { width: 0; height: 0; }` — уже в index.css)

**Tap targets:**
- Кнопки буквы: 36×36px — формально < 44px, но допустимо для secondary navigation (буквы — secondary фильтр к search и chips). Вертикальный padding контейнера `py-2` добавляет 8px сверху и снизу, фактический tap area ≈ 36px high.
- Для увеличения можно сделать `h-[40px]` — тогда ровно 40px. Решение Developer'у.

**Sticky-поведение:**
- Индекс НЕ sticky. Он остаётся в потоке страницы и прокручивается вместе с контентом.
- Причина: sticky потребует сложного z-index управления с action bar. Пользователь и без sticky легко дотягивается до индекса, прокрутив наверх.
