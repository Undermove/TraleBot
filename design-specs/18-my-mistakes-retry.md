# «Мои ошибки» — режим точечного повторения

**Задача:** ROADMAP.md → задача #18
**Статус:** ready

---

## Обучающий элемент

**Грузинская фраза:** `ჩემი შეცდომები` (chemi shecdomébi) — «мои ошибки».

Кнопка появляется ТОЛЬКО в момент провала урока — самый эмоциональный момент сессии. Именно эта контекстуальная привязка делает фразу запоминаемой: пользователь видит «ჩემი შეცდომები» десятки раз в связке «что-то пошло не так → я это исправляю».

**Reveal-момент:**
- «ჩემი» (моя/мой) — притяжательное местоимение 1 л. ед. ч. — встречается в модуле «Местоимения» (Фаза 3.1) и в словарных упражнениях
- Когда пользователь видит «ჩემი» в грамматическом упражнении, он вспоминает кнопку разбора ошибок: «а, это то самое слово» — момент узнавания

**Паттерн:** привычный UI-элемент (кнопка действия) → грузинский ярлык под кнопкой → reveal в модуле местоимений.

## Экраны и состояния

### Экран: Result — обновлённый action bar

Текущий action bar при ошибках (не-словарный модуль, `!isPerfect`):
```
[ к карте уроков →  ]  ← primary (navy)
[ пройти ещё раз    ]  ← ghost
```

**Обновлённый action bar — при наличии ошибок (`correct < total` && `moduleId !== 'vocabulary'`):**

```
┌────────────────────────────────────────────┐
│  [ Повторить ошибки (N)   ]                │  ← ruby fill, 52px
│  ჩემი შეცდომები                            │  ← Georgian label, 10px, centered
│                                            │
│  [ к карте уроков →  ]                     │  ← ghost, 48px
└────────────────────────────────────────────┘
```

**Кнопка «Повторить ошибки (N)»:**
- Цвет: ruby fill (`bg-ruby text-cream border-jewelInk`) — тот же цвет, что у неправильного ответа в Practice, семантически последовательно
- Соответствует Button `variant="green"` (который сейчас маппится на `bg-ruby text-cream`)
- Размер: полная ширина, высота 52px (`jewel-btn` стандарт)
- Текст: `Повторить ошибки (N)` — N это количество неправильных ответов, `text-[15px] font-extrabold`
- Offset shadow: `3px 3px 0 #15100A`; при нажатии: притапливание 3px (стандартный `jewel-btn`)

**Georgian-лейбл под кнопкой:**
- Текст: `ჩემი შეცდომები`
- Шрифт: `font-geo text-[10px] font-bold`
- Цвет: `text-ruby/60` (ruby с 60% opacity — тихий, но узнаваемый)
- Выравнивание: `text-center`
- Отступы: `mt-1.5 mb-3`

**Поведение:**
- Тап → navigate к `{ kind: 'practice-mistakes', moduleId, lessonId, wrongQuestions }`
- `wrongQuestions: QuizQuestion[]` — вопросы, на которые пользователь ответил неправильно в этой сессии, передаются через Screen-state
- Кнопка «к карте уроков» остаётся как вторичное действие
- Кнопка «пройти ещё раз» при наличии ошибок **убирается** — не конкурирует с целевым разбором ошибок

**Полная таблица состояний action bar в Result:**

| Условие | Action bar |
|---------|------------|
| isPerfect + есть nextLesson | «следующий урок →» (green) + «к карте уроков» (ghost) |
| isPerfect + нет nextLesson | «к карте уроков» (primary) + «пройти ещё раз» (ghost) |
| !isPerfect + есть ошибки + не словарь | «Повторить ошибки (N)» (ruby/green) + Georgian label + «к карте уроков» (ghost) |
| Словарный квиз (любой результат) | «к словарю →» (primary) + «на главную» (ghost) |

### Экран: PracticeMistakes — квиз по ошибкам

Отдельный экран, архитектурно идентичен Practice, но с предзагруженными вопросами (без API-запроса).

**Layout:**

```
┌────────────────────────────────────────────┐
│ [═══ килим-полоса ═══════════════════════] │  sticky top
│                                            │
│  [×]  [░░░░░▓▓▓░░░░░░░░░]  1/3          │  header
│                                            │
│  ── разбор ошибок ──                       │  mn-eyebrow (T6)
│  Вопрос № 1                                │  T4, font-extrabold
│                                            │
│  «Как переводится ___?»                    │  T3, question text
│                                            │
│  ┌──────────────────────────────────────┐  │
│  │ a  Вариант ответа A                  │  │  jewel-tile, 52px+
│  └──────────────────────────────────────┘  │
│  ... 4 варианта                            │
│                                            │
│  ┌──────────────────────────────────────┐  │
│  │ ещё раз — explanation                │  │  ruby feedback (если неверно)
│  └──────────────────────────────────────┘  │  navy feedback (если верно)
│                                            │
└────────────────────────────────────────────┘
┌────────────────────────────────────────────┐
│  [ проверить ]  /  [ дальше → ]            │  fixed action bar
└────────────────────────────────────────────┘
```

**Отличия от Practice:**

| Элемент | Practice | PracticeMistakes |
|---------|----------|------------------|
| Загрузка вопросов | API (`lessonQuestions`) | Из props (нет запроса) |
| Progress bar цвет | `bg-navy` | `bg-ruby` |
| Eyebrow | `вопрос № N · грузинское число` | `разбор ошибок` |
| Georgian numerals | Показываются (обучение числительным) | Не показываются (фокус на ошибках) |
| Опция в состоянии `selected` | navy fill | ruby fill |
| Loading state | Есть (LoaderLetter) | Нет (вопросы уже есть) |

**Прогресс-бар:**
- Ruby fill `bg-ruby` — визуальный маркер «режим исправления»
- Прогрессирует от 0 до 100% по мере ответов на все ошибочные вопросы

**Выбор варианта (answering phase):**
- Выбранный вариант: `bg-ruby border-jewelInk text-cream` (вместо navy в обычном Practice)
- Правильный ответ после проверки: по-прежнему `bg-navy border-jewelInk text-cream` (правильное = navy — неизменный семантический якорь)
- Неправильный: ruby + уменьшенная тень (как в Practice)

**Закрытие (×):**
- Тап → navigate к `{ kind: 'module', moduleId }` — карта уроков (не возвращает к Result)
- Telegram BackButton: также к `module`

**Завершение:**
- После последнего вопроса → navigate к `{ kind: 'mistakes-result', moduleId, lessonId, corrected, total, remainingWrong }`
- `corrected` = кол-во правильных ответов в этой попытке
- `total` = общее кол-во ошибок из предыдущего прохода
- `remainingWrong` = вопросы, на которые снова ответили неверно (для возможного повтора)

### Экран: MistakesResult — итог разбора ошибок

Лёгкий экран, структурно похожий на Result, но минималистичнее.

**Layout:**

```
┌────────────────────────────────────────────┐
│ [═══ килим-полоса ═══════════════════════] │
│                                            │
│         [ Бомбора 140px ]                  │
│         [ Stamp overlay ]                  │
│                                            │
│  ── разбор ошибок ──                       │  mn-eyebrow
│  Все ошибки исправлены!                    │  T1, extrabold (если 100%)
│  / Почти разобрались                       │  T1 (если ≥50%)
│  / Ещё поработаем                          │  T1 (если <50%)
│                                            │
│  ┌──────────┐                              │
│  │исправлено│  StatTile (один, по центру)  │  jewel-tile
│  │  2 / 3   │                              │
│  └──────────┘                              │
│                                            │
│  [comment text]                            │  T5, jewelInk-soft
│                                            │
└────────────────────────────────────────────┘
┌────────────────────────────────────────────┐
│  [ к карте уроков →  ]                     │  primary
│  [ повторить оставшиеся (N) ]              │  ghost (только если remainingWrong.length > 0)
└────────────────────────────────────────────┘
```

**Mascot mood и Stamp:**

| Исправлено | Mood | Stamp (GEO) | Stamp (RU) | Цвет штампа |
|------------|------|-------------|------------|-------------|
| 100% | `cheer` | `ბრავო` | «браво» | ruby |
| ≥50% | `happy` | `კარგი` | «хорошо» | ruby |
| <50% | `think` | `ცადე` | «попробуй» | ruby |

Штамп всегда ruby (не gold и не navy) — в этом экране он маркирует завершение ошибочного цикла.

**Заголовок и комментарий:**

| Исправлено | Title | Comment |
|------------|-------|---------|
| 100% | «Все ошибки исправлены!» | «Блокнот стал немного полнее.» |
| ≥50% | «Почти разобрались» | «Повтори оставшееся — это ускорит запоминание.» |
| <50% | «Ещё поработаем» | «Трудные вопросы — перечитай теорию и попробуй снова.» |

**StatTile:**
- Один тайл, выровнен по левому краю (или по центру, если один)
- Label: `исправлено`, Value: `N / M`, Accent: `ruby`
- Размер: `max-w-[140px]` (не растягивать на всю ширину — это не 3-колонная сетка)

**Конфетти:** отсутствует. Даже при 100% в режиме разбора ошибок конфетти не запускается — это вторичный успех, а не завершение урока.

**Action bar:**
- При `corrected === total`: только «к карте уроков →» (primary)
- При `corrected < total`: «к карте уроков →» (primary) + «повторить оставшиеся (N)» (ghost)
  - Ghost кнопка: navigate к новому `practice-mistakes` с `remainingWrong` как новым набором вопросов

### Компоненты (изменения)

#### Button — новый вариант `'ruby'` (опционально)

Текущий `variant="green"` уже маппится на `bg-ruby text-cream` — Developer может переиспользовать его ИЛИ добавить явный `variant="ruby"` для семантической ясности. Визуальный результат одинаков:

```
bg-ruby text-cream border-jewelInk
box-shadow: 3px 3px 0 #15100A
active: translateX(1.5px) translateY(1.5px), shadow: 1px 1px 0 #15100A
```

Если добавляется новый вариант — назвать `'ruby'` (не `'error'` и не `'danger'`): семантика «исправление», не «опасность».

#### Result — изменение Props

Добавляется поле:
```typescript
wrongQuestions?: QuizQuestion[]
// Массив вопросов, на которые пользователь дал неправильный ответ в текущей сессии Practice.
// Передаётся через Screen state: { kind: 'result', ..., wrongQuestions }
```

#### Practice — изменение данных

Добавляется внутренний стейт:

При завершении (navigate к result) — передаётся в Screen state.

## Поток данных

```
Practice
  ├── добавить: wrongQuestions: QuizQuestion[] = []
  ├── при неверном ответе: wrongQuestions.push(current)
  └── navigate({ kind: 'result', ..., wrongQuestions })

Result
  ├── получает wrongQuestions? через props
  ├── if wrongQuestions?.length > 0 && moduleId !== 'vocabulary':
  │   └── рендерит кнопку «Повторить ошибки (N)»
  │       └── onClick: navigate({ kind: 'practice-mistakes', moduleId, lessonId, wrongQuestions })
  └── иначе — текущее поведение без изменений

PracticeMistakes (новый экран)
  ├── получает wrongQuestions: QuizQuestion[] через props (без API-запроса)
  ├── добавить: missedAgain: QuizQuestion[] = []
  ├── при неверном ответе: missedAgain.push(current)
  └── navigate({ kind: 'mistakes-result', moduleId, lessonId,
                 corrected: (wrongQuestions.length - missedAgain.length),
                 total: wrongQuestions.length,
                 remainingWrong: missedAgain })

MistakesResult (новый экран)
  ├── показывает итог
  ├── «к карте уроков» → navigate({ kind: 'module', moduleId })
  └── «повторить оставшиеся (N)» → navigate({ kind: 'practice-mistakes', ..., wrongQuestions: remainingWrong })

App.tsx BackButton:
  - 'practice-mistakes' → { kind: 'module', moduleId }
  - 'mistakes-result'   → { kind: 'module', moduleId }
```

**Персистентность:** только session-state. После перехода к `module` или `dashboard` данные теряются — это корректное поведение.

- Не использовать «провал», «поражение» — только нейтральное «ошибки»

- Georgian метки — строго отдельные строки, не вставляются в русский текст
- Не использовать «режим» или «функция» — просто называть действие
