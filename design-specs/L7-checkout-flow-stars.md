## Обучающий элемент

**Грузинское слово:** `ვარსკვლავი` (varskvlavi) — звезда.

Telegram оплата называется «Stars» (звёзды). На paywall-экране под ценой стоит грузинский субтайтл `ვარსკვლავი · звезда`. Пользователь видит это слово каждый раз при взгляде на цену.

**Reveal-момент:** при прохождении урока «Лексика / Природа» или встрече слова `ვარსკვლავი` в словаре пользователь узнаёт: «это же то слово с экрана оплаты!» — мнемонический якорь через финансовый жест.

**Паттерн:** привычный элемент (кнопка оплаты с ⭐) → грузинское слово-субтайтл → reveal в учебном контексте.

**Trigger-варианты:**

|---------|-----------|---------------------|-----------|

| `vocabulary_limit` | Словарь заполнен | ლექსიკონი · 50/50 | ✓ Словарь без ограничений — добавляй сколько угодно |

### Экран 2: Telegram Invoice (нативный, вне мини-аппа)

Открывается через `Telegram.WebApp.openInvoice(url)`. Это нативный UI Telegram — мини-апп не управляет его внешним видом. Мини-апп переходит в фоновый режим, пока открыт инвойс.

**Что важно для дизайнера:**
- Название товара (title): «TraleBot Pro» — строго 1–32 символа
- Описание (description): «Все модули грузинского языка. Словарь без лимита.» — до 255 символов
- Фото товара: логотип Бомборы (Mascot) 640×360px — задаётся в backend-конфигурации

**Callback-события (обрабатываются ProPaywall):**

| `invoice_closed.status` | Действие |
|------------------------|----------|
| `paid` | Закрыть paywall → вызвать onPurchaseSuccess() |
| `cancelled` | Вернуться в состояние `default` |
| `pending` | Вернуться в состояние `default` |
| `failed` | Показать состояние `error` |

### Экран 3: Success Toast (после оплаты)

Появляется поверх Dashboard или Profile на 3.5 секунды, затем автоматически скрывается.

**Layout:**
```
┌────────────────────────────────────────┐
│  ★ Добро пожаловать в Pro!             │  ← T3, extrabold, ink
│  ყველა მოდული ღიაა                     │  ← T6, geo font, bold, ink/70
└────────────────────────────────────────┘
```

**Стиль:** gold (#F5B820) background, 2px border-b jewelInk. Прижат к верху экрана (учитывает safe-area-top). Анимация: slideDown 250ms ease-out.

**Что происходит параллельно:**
- `isPro` флаг переключается в App state → все Pro-locked модули на Dashboard мгновенно разблокируются (ProBadge исчезает, opacity 60% → 100%)
- Если пользователь находится на Profile — RefundButton появляется вместо CTA «Выбрать тариф»

---

### Экран 4: Pro-состояние на Dashboard

После успешной оплаты Pro-locked модули переходят в активное состояние.

**До оплаты (free user):**
- Иконка модуля: opacity 60%
- Правый угол тайла: `★ Про` (ProBadge, gold pill)
- Тап → открывает ProPaywall с trigger='module'

**После оплаты (Pro user):**
- Иконка: opacity 100%
- ProBadge: скрыт
- Тап → нормальный переход в модуль

**Анимация разблокировки** (желательное улучшение, не обязательное для launch):
- При переходе free → Pro: плавное появление opacity 60%→100% за 300ms для всех затронутых тайлов

### Экран 5: Pro-статус и возврат на Profile

**Layout (Pro user):**
```
┌──────────────────────────────────────┐
│  ★ Pro-подписка активна              │  ← T4, gold star, ink text
│  с [дата покупки]                    │  ← T6, ink-mid
│                                      │
│  [ Вернуть платёж ]                  │  ← text link, ink-mid, T5
└──────────────────────────────────────┘
```

**RefundButton — состояния:**

| Состояние | UI |
|-----------|-----|
| `idle` | Ссылка «Вернуть платёж» (T5, ink-mid) |
| `confirming` | jewel-tile: «Вернуть звёзды?» + описание + [Вернуть] (ruby) + [Отмена] (cream) |
| `pending` | LoaderLetter в tile |
| `done` | «✓ Возврат оформлен» — jewel-tile, затем через 1.2с isPro → false |
| `error` | Сообщение ошибки + ссылка «Попробовать снова» |

**Важно:** возврат доступен 21 день — требование Telegram. По истечении 21 дня кнопка «Вернуть» скрывается (backend валидирует, фронтенд — если backend вернул ошибку → error state).

## Адаптивность

- Bottom sheet: `max-w-[480px] mx-auto` — не растягивается на широких экранах
- На 375px тарифные кнопки занимают полную ширину (flex-col) — читаемо
- CTA-кнопка: `min-h-[52px]` — превышает минимальный tap target 44px
- «Нет, пока нет»: `min-h-[44px]` — граничное значение, ок
- Все drag-targets ≥ 44px
