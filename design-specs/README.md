# Design Specs — Бомбора

Здесь живут дизайн-спецификации для фич мини-аппа. Каждая спека создаётся Designer-агентом и является обязательным входом для Developer-агента.

## Процесс

1. Product-агент создаёт задачу `[idea]` в ROADMAP.md
2. Designer-агент берёт задачу, создаёт спеку здесь, статус → `[designed]`
3. Developer-агент реализует по спеке, статус → `[dev]` → `[review]`
4. Tech Lead ревьюит соответствие спеке

## Именование файлов

`{номер-задачи}-{slug}.md`

Примеры:
- `11-progressive-unlock.md`
- `14-treat-shop.md`

## Статусы спеки

- `draft` — в работе, не готова для реализации
- `ready` — можно реализовывать
- `implemented` — реализована, PR смёрджен
