# Georgian-only: бот для изучения грузинского

**Задача:** ROADMAP.md → задача #21
**Статус:** ready

---

## Обучающий элемент

**Первое слово — `გამარჯობა`.**

Новый пользователь встречает его в самом первом сообщении бота — ещё до того, как откроет мини-апп, до первого урока, до всего. Это слово лежит в долгосрочной памяти с момента `t=0`.

**Reveal-момент:** когда пользователь доходит до модуля «Приветствия» (фаза 2.1) и видит `გამარჯობა` — он узнаёт: «Я это видел ещё в самом начале, это же "привет"!» Двойной дофамин: узнавание + гордость («я это запомнил, не учив специально»).

**Паттерн:** привычный элемент (приветствие от бота) → грузинский контекст (`გამარჯობა`) → reveal в учебном модуле.

---

## Режимы работы

| Режим | Условие | Поведение |
|-------|---------|-----------|
| **Georgian-mode** | все пользователи | Язык = Georgian по умолчанию, без выбора, без English-кнопок |
| **Owner-mode** | `TelegramId == OWNER_TELEGRAM_ID` | Полный доступ: смена языка, кнопки Wooordhunt/Reverso Context |

Проверка owner-режима — через env-переменную `OWNER_TELEGRAM_ID`. Если переменная не задана — owner-режима нет.

## Экраны и состояния

### Экран: /start — новый пользователь

**Текущий флоу (убираем):**
1. Приветственное сообщение с LanguageKeyboard («Английский / Грузинский»)
2. Выбор языка → SetInitialLanguage → меню

**Новый флоу:**
1. Приветственное сообщение → система автоматически ставит `CurrentLanguage = Georgian`
2. Сообщение уже содержит CTA-кнопку «Показать меню»

**Копирайт нового сообщения:**

```
გამარჯობა, {UserName}! 👋

Меня зовут TraleBot — я помогаю учить грузинский язык.

Только что ты узнал своё первое слово:
გამარჯობა — «привет» по-грузински.

Со мной ты можешь:
• Переводить слова и пополнять словарь
• Учиться через мини-апп «🐶 Бомбора»

Напиши мне любое слово или фразу — переведу и добавлю в словарь.
```

**Клавиатура под сообщением (InlineKeyboard):**

```
[ 📋 Меню ]
```

Одна кнопка. Нет выбора языка. Нет объяснений про «мультиязыковой словарь».

**Состояния:**
- Default (новый пользователь): описано выше
- Пользователь с shared quiz (`/start {quizId}`): поведение не меняется — логика shared quiz остаётся прежней

### Экран: Результат перевода — обычный пользователь

**Убираем** кнопки Wooordhunt и Reverso Context. Они появлялись только при `CurrentLanguage == English` — поскольку все пользователи будут в Georgian-режиме, этот блок больше не нужен.

**Текущий состав клавиатуры перевода:**
1. ❌ Не добавлять в словарь
2. Перевод Wooordhunt ← **УБРАТЬ** (English-only)
3. Перевод Reverso Context ← **УБРАТЬ** (English-only)
4. 🔄 Перевести на другой язык
5. 📋 Меню

**Новый состав:**

```
[ ❌ Не добавлять в словарь        ]
[ 🔄 Перевести на другой язык      ]
[ 📋 Меню                          ]
```

3 строки вместо 5. Чище, меньше ссылок на сторонние ресурсы.

**Для владельца в English-режиме:** кнопки Wooordhunt и Reverso Context остаются (поведение не меняется).

## Изменения в командах и сообщениях

### Убрать из флоу

| Команда / элемент | Действие |
|-------------------|----------|
| `LanguageKeyboard.GetLanguageKeyboard` в `StartCommand` | Убрать — не показываем выбор языка |
| `SetInitialLanguage` bot command | Деактивировать для обычных пользователей (или оставить как no-op) |
| Кнопка `ChangeCurrentLanguageMenu` в `MenuKeyboard` | Скрыть для не-владельца |
| Кнопки Wooordhunt и Reverso Context | Скрыть для не-владельца (или когда `CurrentLanguage != English`) |

### Изменить

| Что | Как |
|-----|-----|
| `CreateUser` handler | `CurrentLanguage = Language.Georgian` вместо `Language.English` |
| `StartCommand.Execute` | Новый приветственный текст, CTA-кнопка «Меню» вместо LanguageKeyboard |
| `MenuKeyboard.GetMenuKeyboard` | Принять `isOwner: bool` — скрывает/показывает кнопку смены языка |
| `TranslationKeyboard` | Условие `Language.English` для Wooordhunt/Reverso заменить на `isOwner && Language.English` |

## Технические заметки для Developer

**Owner detection:**
- Env-переменная `OWNER_TELEGRAM_ID` (long)
- Если не задана или 0 — owner-mode отключён для всех
- Проверяется в `BotConfiguration` и прокидывается в команды через DI или прямую проверку в хендлере

**Изменения в domain:**
- `CreateUser` handler: `CurrentLanguage = Language.Georgian`
- Нет изменений в схеме БД — `Language` enum остаётся, просто меняется дефолт

**Изменения в `StartCommand`:**
- Убрать вызов `LanguageKeyboard.GetLanguageKeyboard`
- Новый текст приветствия
- Inline-кнопка `[ 📋 Меню ]` → callback `CommandNames.Menu`
- Новый user получает Georgian сразу через `CreateUser` (дефолт изменён)

**Изменения в `MenuKeyboard`:**
- Добавить параметр `bool isOwner = false`
- Строка `"Сменить язык словаря: ..."` — только если `isOwner`

**Изменения в `TranslationKeyboard`:**
- Условие для Wooordhunt/Reverso Context: `isOwner && currentLanguage == Language.English`

**Как передать `isOwner`:**
- `BotConfiguration` добавляет поле `long OwnerTelegramId`
- В `TelegramRequest` или в каждом хендлере: `bool isOwner = request.UserTelegramId == botConfig.OwnerTelegramId`
