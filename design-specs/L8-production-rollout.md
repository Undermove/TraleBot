## Обучающий элемент

Сам ролаут — точка контакта: пользователь видит новую кнопку в чате и первый раз открывает мини-апп. Обучение начинается не с урока, а с **момента открытия**:

- Кнопка меню: **«🏔 Бомбора»** — имя маскота на русском, лаконично, без перевода; Georgian script присутствует на первом же экране Onboarding (`გამარჯობა!` над заголовком)
- **Reveal-момент для существующих пользователей:** они привыкли получать переводы в чате — теперь видят, что те же слова живут в «Мой словарь» внутри приложения. «Мои переводы = мой первый урок.»
- Кнопка «🚀 Открыть TraleBot» в inline-keyboard /start уже содержит обучающий подтекст (первое слово — `გამარჯობა` — прямо в приветственном сообщении). Это сохранить без изменений.

## Экраны и состояния

### Экран 1 — Кнопка меню (Telegram UI)

Это не экран мини-аппа, а кнопка слева от поля ввода текста в чате.

**Текущее состояние (MiniAppEnabled=false):**
```
[ ⋮ Меню ] [ Введите сообщение... ] [→]
```
Стандартный значок команд. Пользователь не видит мини-апп.

**После флипа (MiniAppEnabled=true):**
```
[ 🏔 Бомбора ] [ Введите сообщение... ] [→]
```
WebApp-кнопка. Один тап → открывается мини-апп поверх чата.

**Дизайн-решение: label кнопки**

| Вариант | Текст | Плюсы | Минусы |
|---|---|---|---|
| A (текущий) | `🚀 TraleBot` | Название продукта, узнаваемость | Ракета — не тематически грузинская |
| B (предлагается) | `🏔 Бомбора` | Маскот, теплее, эмодзи-гора = Кавказ | Новые пользователи не знают Бомбору |
| C | `ქ Учить грузинский` | Явный призыв | Длинно, не умещается на узких экранах |

**Выбор: вариант B — `🏔 Бомбора`.** Минус нивелируется тем, что уже первое открытие объясняет маскота. Emoji 🏔 семантически связан с Кавказом — мягкий культурный маркер без этнического акцента.

### Экран 2 — Приветствие существующего пользователя после /start

После флипа существующий бот-пользователь, нажавший `/start`, должен увидеть **обновлённое приветствие**, явно анонсирующее мини-апп. Текущее «возвращательное» сообщение (`გამარჯობა снова...`) слишком нейтральное.

**Состояния:**

**2a. Пользователь уже имеет Georgian-режим (наиболее частый случай):**

```
გამარჯობა снова, {UserName}! 👋

Пока тебя не было — мы открыли мини-апп.
Теперь Бомбора ждёт тебя прямо здесь, в Telegram:
алфавит, грамматика, словарь из твоих переводов, квизы.

Жми «🏔 Бомбора» рядом с полем ввода — или кнопку ниже.

Кстати: გამარჯობა — «привет» по-грузински.
Ты это уже знаешь? 😄
```

Inline-keyboard:
```
[ 🏔 Открыть Бомбору ]    ← WebApp-кнопка (primary)
[ ⋮ Меню в чате ]         ← callback (secondary, одна строка)
```

**2b. Пользователь без Georgian-режима (редкий случай после Georgian-only commit):**
Стандартное «возвращательное» сообщение без изменений — мини-апп ему недоступен до смены языка.

**Примечание реализации (не для дизайна):** разделить ветку `else` в `StartCommand.Execute` по признаку `botConfig.MiniAppEnabled && user.CurrentLanguage == Georgian`.

### Экран 3 — Первое открытие мини-аппа у существующего пользователя

**Проблема:** пользователь уже переводил слова в боте → у него есть `VocabularyEntry` записи. Он открывает мини-апп впервые и видит Onboarding («Начинаю с нуля» / «Уже что-то знаю»).

**Дизайн-решение:** Onboarding остаётся **без изменений**. Причины:
1. Форма «с нуля / уже знаю» не о словаре — она о знании алфавита.
2. Оба пути попадают на одинаковый Dashboard.
3. Уровень (beginner/intermediate) только влияет на начальное состояние разделов Dashboard (collapsed / expanded).

**Однако** — на экране Onboarding нет сигнала «твой словарь уже тут». Добавить **sub-hint** под двумя кнопками:

```
[ ა  Начинаю с нуля     ]
[ ქ  Уже что-то знаю    ]

  Твои переводы из бота уже в «Мой словарь» 📖
  (серый текст, 12px, по центру)
```

Этот hint показывается только если `/api/miniapp/me` вернул `vocabularyCount > 0`. Для новых пользователей без слов — скрыт.

**Компонент:** добавить `vocabularyCount?: number` в props `Onboarding`. Визуально: `font-sans text-[12px] text-jewelInk-hint text-center mt-3` — те же стили что у уже существующего sub-hint «Все уроки будут доступны...».

### Экран 4 — «Мой словарь» с импортированными словами (happy path)

Если существующий пользователь переводил слова → открыл мини-апп → выбрал уровень → попал на Dashboard → нажал «Мой словарь»: он видит **свои слова**. Это должно быть сюрпризом.

**Дизайн-решение:** добавить **первый-раз баннер** в `VocabularyList`, если:
- это первое открытие словаря (localStorage `vocab_first_open` не проставлен)
- `words.length > 0`

**Баннер — `FirstVocabBanner`:**
```
┌─────────────────────────────────────────────────┐
│  📖  Твои переводы уже здесь                    │
│      Все слова, которые ты переводил в боте,    │
│      собраны в один словарь. Хочешь их          │
│      закрепить? → [Начать квиз]                 │
└─────────────────────────────────────────────────┘
```

Стиль: `jewel-tile`, navy border. Кнопка «Начать квиз» — `jewel-btn` (navy). После тапа на кнопку или закрытия — `vocab_first_open = 'seen'` в localStorage, баннер исчезает навсегда.

Reveal-момент: «Мои случайные переводы — это уже мой первый учебник!»

## Runbook флипа (операторская часть — не UI, но необходимо для спеки)

Это не дизайн UI, но даёт Developer-агенту критерий «готово»:

### Шаг 1 — Проверить зависимости
- [ ] L7 checkout test — закрыт (#268)
- [ ] Все миграции применены в staging (проверить `dotnet ef database update`)
- [ ] `MiniAppEnabled=false` в проде сейчас — верифицировать через `/api/miniapp/me` → `miniAppEnabled: false`

### Шаг 2 — Применить изменение
```
# appsettings.Production.json или переменная окружения
BotConfiguration__MiniAppEnabled=true
```
Деплой → бот при старте вызовет `SetChatMenuButtonAsync` с WebApp-кнопкой.

### Шаг 3 — Верификация (< 5 минут)
1. Открыть Telegram, найти бота
2. Убедиться: кнопка **«🏔 Бомбора»** слева от поля ввода
3. Нажать кнопку → мини-апп открылся поверх чата
4. Пройти Onboarding → Dashboard загружен
5. Открыть «Мой словарь» → слова отображаются (если есть)
6. Нажать `/start` → приветственное сообщение обновлённое (с CTA Открыть Бомбору)
7. `/api/miniapp/me` возвращает `miniAppEnabled: true`

### Шаг 4 — Rollback-критерий
Откатить `MiniAppEnabled=false` если в течение 30 минут:
- более 5 пользователей сообщают об ошибке загрузки мини-аппа
- `/api/miniapp` возвращает 5xx
- checkout flow сломан (нет успешных покупок при тестовом заходе)
