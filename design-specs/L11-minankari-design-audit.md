## Обучающий элемент

**Kilim-полоса** (mn-kilim) — 8px-орнамент из ромбов в цветах navy/ruby/gold — продуктовая подпись Minankari. Пользователь видит её на каждом экране как декоративную деталь. **Reveal-момент:** когда проходит раздел «История алфавита» (#33, будущий модуль) и узнаёт, что зигзаг-ромбы отсылают к кавказскому коврoвому орнаменту килим — тот же мотив, что в украшениях мтиульской традиции. До reveal — просто красивая рамка. После — узнаваемый культурный символ.

## Типографическая шкала T1–T6 (формализация)

Существующий код использует произвольные пиксельные размеры (10–44px). Эта спека фиксирует стандарт — все новые экраны и правки должны использовать только эти значения:

| Уровень | Размер | font-weight | Применение |
|---------|--------|-------------|------------|
| T1      | 44px   | 800         | Hero-заголовки (Result «Отлично!», Landing H1) |
| T2      | 28px   | 800         | Экранные заголовки (Onboarding «Привет!») |
| T3      | 20px   | 800         | Значения статистики, крупные числа/метрики |
| T4      | 16px   | 700–800     | Названия модулей, тело кнопок, крупный body |
| T5      | 14px   | 400–600     | Body-текст, описания, подписи |
| T6      | 11px   | 700         | Eyebrow-метки (mn-eyebrow), служебный текст |

**Нестандартные размеры в текущем коде → что делать:**
- `12px` → T6 (11px) или T5 (14px) по контексту
- `13px` → T5 (14px) или T6 (11px)
- `15px` → T5 (14px) или T4 (16px)
- `18px` → T4 (16px) (если заголовок) или T3 (20px)
- `24px` → T3 (20px) или T2 (28px)

Не нужно приводить весь код разом. Применять при ближайшем касании экрана.

## Принцип «один первичный CTA на экран»

**Правило:** не более одной ярко-окрашенной кнопки в действующей зоне экрана. Иерархия вариантов Button:

```
primary (navy)  >  green (ruby)  >  blue (gold)  >  ghost (cream)
     ↑                   ↑                ↑               ↑
  основное           успех/ретрай    тёплый вторичный   нейтральный
  действие           (красный)       (жёлтый)           фон
```

**Допустимо:** primary + ghost, green + ghost, green + ghost — один яркий, один нейтральный.  
**Недопустимо:** primary + green, primary + blue — два ярких одновременно.

### Экран: Result

**Статус kilim:** ❌ отсутствует и сверху, и снизу.

**Правка kilim:**

```tsx
// В начале JSX, сразу после <Header variant="result" .../>:
<div className="mn-kilim" />

// В конце JSX, после action-bar (перед последним </div>):
<div className="mn-kilim opacity-70" />
```

**Статус CTA-иерархии:**

| Состояние | Кнопки | Соответствие |
|-----------|--------|--------------|
| Ошибки есть | `green` «Повторить» + `ghost` «к карте» | ✅ |
| 100% + след. урок | `green` «следующий урок» + `ghost` «к карте» | ✅ |
| Нет след. урока | `primary` «к карте» + `ghost` «ещё раз» | ✅ |
| Словарь | `primary` «к словарю» + `ghost` «на главную» | ✅ |

Статистические плитки (navy/ruby/gold) — не CTA, несут смысловую нагрузку (правильно / XP / точность). Три разных акцентных цвета на одном экране допустимы — каждый на своём элементе, не перемешиваются.

**Визуальный баг — tap target:**  
Нет явных мелких кнопок — плитки-статистики не кликабельны. ✅

### Экран: Dashboard

**Статус kilim:** ✅ Верхняя (в DashboardTopBar), нижняя (строка 518).

**Статус CTA-иерархии:** ⚠️ Потенциальный конфликт.

Hero-зона содержит:
1. Кнопка «Покормить · ⭐ XP» — `bg-gold text-jewelInk` (gold filled, визуально primary)
2. Suggestion tile — `jewel-tile jewel-pressable` с ruby-иконкой (сильный акцент)
3. Модульные плитки «старт» — `jewel-tile jewel-pressable` (navigation)

Основное действие пользователя — **начать/продолжить урок** (тап по модулю). Кнопка «Покормить» — retention-механика, вторичное действие.

**Правка — визуальная иерархия «Покормить»:**

Кнопку «Покормить» перевести из gold-filled в ghost-стиль:

```tsx
// Было:
className="px-4 py-2 rounded-xl font-sans text-[13px] font-extrabold bg-gold text-jewelInk border-[1.5px] border-jewelInk active:scale-95 transition-transform flex items-center gap-2"

// Стало:
className="px-4 py-3 rounded-xl font-sans text-[13px] font-semibold bg-cream-tile text-jewelInk border-[1.5px] border-jewelInk/60 active:scale-95 transition-transform flex items-center gap-2"
```

Изменения:
- `bg-gold` → `bg-cream-tile` (нейтральный фон, не конкурирует с модулями)
- `font-extrabold` → `font-semibold` (снижена визуальная громкость)
- `py-2` → `py-3` (tap target: 8px padding × 2 + 13px line-height → ≥ 44px ✅)
- `border-jewelInk` → `border-jewelInk/60` (граница мягче)

**Альтернатива:** если кнопка «Покормить» должна оставаться gold (продукт считает это важным retention-сигналом) — вынести её из hero в отдельную секцию под маскотом, чтобы пространственно отделить от navigation-тайлов.

**Tap target — текущее нарушение:**  
`py-2` = ~8px padding top+bottom, строка `text-[13px]` ≈ 20px → итого ~36px. **< 44px, нарушение.** Обязательная правка: минимум `py-3`.

## Адаптивность (375px)

**Dashboard hero (критично):** маскот + строка статуса + кнопка «Покормить» + suggestion tile — весь блок должен влезать в ~260px высоты без вертикального скролла до первого launch-модуля. Проверить при sated Бомборе (mood влияет на размер).

**Result:** конфетти-слой — `overflow-hidden` уже есть ✅. Stat-tiles в `grid-cols-3` — при 375px каждая ячейка ≈ 110px, достаточно для `+120 xp`. ✅

**Onboarding:** два jewel-tile `py-5` = 56px минимум + gap + текст + маскот ≈ 420px. На 375×667 (iPhone SE) — умещается без скролла. ✅

**LessonTheory action bar:** фиксированный bottom bar — проверить что при mn-kilim снизу не перекрывает контент (добавить `pb-[8px]` к safe-area или разместить kilim внутри контейнера, а не вне).
