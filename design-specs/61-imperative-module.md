**Задача:** ROADMAP.md → задача #61 · GitHub issue #385  
**Статус:** ready

---

## Контекст проблемы

Phase 4 вводит шесть и более императивных форм как «замороженные фразы»: «შეხვიდე!» (войди), «გამოდი!» (выходи), «დაჯექი!» (садись). Ученик заучивает список — но система за ними не объяснена.

Когда нужно скомандовать «открой», «напиши», «купи» — ученик не может. Систематический пробел в A2→B1: любой турист даёт указания таксисту («поверни налево»), официанту («принеси счёт»), другу («подожди меня»). Без знания системы императива ученик молчит или использует инфинитив, что звучит грубо.

---

## Обучающий элемент

### Reveal-момент: основа аориста = основа императива

Ученик уже знает аорист из пройденного модуля:
- «я написал» → **და-ვ-წერ-ე** (преверб + -ვ- + корень + -ე)

Открытие: **убери преверб и -ვ-е — останется императив**:

```
და-ვ-წერ-ე  →  წერ-ე  →  «напиши!»
   ↑ убрать      ↑ это уже повелительное
```

Та же аорист-основа, знакомая из прошлого, — только без личных маркеров. Это не исключение, это система: ученик **уже знает** как строить команды, просто не знал об этом.

**«Wow»-момент:** «Значит, я мог командовать с тех пор, как выучил аорист? Я просто не знал формулы!»

### Вторичные обучающие моменты

1. **Отрицательный императив** — «ნუ» + имперфектная основа: система «запрета» использует не аорист, а настоящее/имперфект. Это закрепляет разницу аспектов: аорист = завершённость (сделай), имперфект-основа = процесс (не делай [вообще]).

2. **Вежливость через -ეთ** — тот же суффикс, что в 2л.мн. настоящего («вы пишете» = წერთ). Reveal: вежливость = обращение к множеству. Грузинская логика вежливости на поверхности.

### Экран: L1 — «Напиши!» (2л.ед. от Class 1 глаголов)

**Цель:** Объяснить формулу. Показать связь с аористом.

**Теория (блоки ContentProvider):**

1. **Paragraph (hook):**  
   «Ты уже знаешь "я написал" — **და-ვ-წერ-ე**. Убери преверб и личные суффиксы — останется **წერ-ე**. Это и есть "напиши!"»

2. **List (парадигма "писать", императив 2л.ед.):**
   - **დაწერე** — напиши! (преверб + корень + -ე)
   - **შეჭამე** — съешь! (преверб შე- + корень)
   - **გააკეთე** — сделай! (преверб გა- + корень)
   - **წაიკითხე** — прочитай! (преверб წა- + корень)

3. **Paragraph (формула):**  
   «Формула: преверб (из аориста) + корень + -ე. Тот же преверб, что в будущем и аористе — только без -ვ- в начале и без -ვ-е на конце.»

4. **Example (контраст: аорист vs императив):**
   ```
   Аорист 1л.:   და-ვ-წერ-ე  →  «я написал»
   Императив 2л.: და-  წერ-ე  →  «напиши!»
                      ↑ убрать -ვ-
   ```

5. **Paragraph (scoping — что НЕ входит в L1):**  
   «В этом уроке — Class 1 глаголы с одним правилом. VoM-глаголы и Class 3 — особая история, выучим позже.»

**Квизы (5 вопросов, `questions1.json`):**
- `imp1_q01`: «Как сказать "напиши" по-грузински?» → дать/написать/ прочитать → და**წე**რე
- `imp1_q02`: «Переведи: "შეჭამე"» → [съешь / съел / ешь сейчас]
- `imp1_q03`: «Какая форма — императив?» → [დავწერე / **დაწერე** / წერს]
- `imp1_q04`: type-answer: «Напишите по-грузински "прочитай!"» → წაიკითხე
- `imp1_q05`: «Что общего у императива и аориста?» → [один корень / один преверб / оба ответа верны ✓]

### Экран: L2 — «Не пиши!» (отрицательный императив)

**Цель:** Паттерн «ნუ + имперфектная основа». Понять, почему аспект меняется.

**Теория (блоки ContentProvider):**

1. **Paragraph (hook):**  
   «Запрет в грузинском — не просто "нет" + императив. Для запрета нужна частица **ნუ** и другая основа глагола.»

2. **List (контраст положительного и отрицательного):**
   - **დაწერე** — напиши!  
   - **ნუ წერ** — не пиши! (ნუ + основа настоящего, без преверба)
   - **შეჭამე** — съешь!  
   - **ნუ ჭამ** — не ешь!

3. **Paragraph (объяснение):**  
   «Аорист = завершённое действие → "сделай (это)". Имперфект-основа = незавершённый процесс → "не делай [вообще]". Грузинский запрет запрещает процесс, не результат.»

4. **Example (мнемоника):**
   ```
   ნუ + [основа настоящего] = запрет
   ნუ წერ!   — не пиши!
   ნუ ჭამ!   — не ешь!
   ნუ დარეკ! — не звони!
   ```

5. **Paragraph (важно — ошибка учеников):**  
   «Нельзя сказать "ნუ დაწერე" — это звучит как незаконченный запрет. Всегда: ნუ + форма без преверба.»

**Квизы (4 вопроса, `questions2.json`):**
- `imp2_q01`: «Как сказать "не ешь!"» → [ნუ შეჭამე / **ნუ ჭამ** / ნუ ჭამე]
- `imp2_q02`: «Что использует грузинский для запрета?» → [ნუ + аорист / **ნუ + основа наст.** / ნუ + инфинитив]
- `imp2_q03`: type-answer: «Напишите "не пиши!"» → ნუ წერ
- `imp2_q04`: «Переведи: "ნუ დარეკ!"» → [не позвони / **не звони** / не звал]

### Экран: L3 — «Напишите!» (вежливый / 2л.мн. императив)

**Цель:** Суффикс -ეთ превращает команду в просьбу / вежливое обращение.

**Теория (блоки ContentProvider):**

1. **Paragraph (hook):**  
   «В грузинском вежливость = обращение ко многим. Добавь **-ეთ** к императиву — и «напиши» становится «напишите» (вежливо).»

2. **List (пары: неформальное / вежливое):**
   - **დაწერე** → **დაწერეთ** — напишите / напиши (вежливо)
   - **შეჭამე** → **შეჭამეთ** — съешьте / пожалуйста, съешьте
   - **დაჯექი** → **დაჯექით** — садитесь (к группе / вежливо к одному)

3. **Paragraph (reveal — связь с настоящим временем):**  
   «Тот же суффикс -ეთ/-თ ты видел в настоящем: **წერ-თ** (вы пишете). Грузинский использует один инструмент и для "вы" в глаголе, и для вежливости. Не новое правило — расширение того, что ты знаешь.»

4. **Example (ситуационный блок):**
   ```
   Таксисту:   «გაჩерეთ!» — остановитесь, пожалуйста
   Официанту:  «მოიტანეთ ანგარიში» — принесите счёт
   Другу:      «მოდი!» → «მოდით!» — идёмте!
   ```

5. **Paragraph (отрицательный вежливый):**  
   «Запрет вежливо: **ნუ** + форма с -ეთ: **ნუ წეROTEთ** → не пишите. Правило ნუ сохраняется.»

**Квизы (4 вопроса, `questions3.json`):**
- `imp3_q01`: «Как вежливо попросить "напишите"?» → [დაწერე / **დაწერეთ** / წერეთ]
- `imp3_q02`: «Что добавляет -ეთ к императиву?» → [вежливость / число / и то, и другое ✓]
- `imp3_q03`: «Переведи: "დაჯექით"» → [сядь / **садитесь** / сидите сейчас]
- `imp3_q04`: type-answer: «Как сказать "не пишите!" вежливо?» → ნუ წეROTEთ

*(Итого 13 вопросов — более минимума в 12)*

## Edge cases

1. **Аорист не пройден** — AdvisoryBanner при первом входе. Модуль не блокируется.
2. **Глаголы VoM/Class 3** — явно выведены за скоуп в L1 theory блоком «Scoping». Нет примеров с ვიდიხარ/მინდა.
3. **Только Class 1 глаголы в квизах** — გადადი (VoM), მინდა (Class 3) в дистракторах не используются.
4. **Регрессия** — `FeatureCatalogCoverageTests` должен видеть `imperative` в каталоге модулей и зелёным после добавления.
5. **Зависимость от FutureTense** — L3 использует -ეთ, знакомый по настоящему времени. Явная ссылка в теории не зависит от порядка прохождения — она справочная.
6. **Пустой словарь** — модуль не зависит от словарных данных, работает без vocabulary.
