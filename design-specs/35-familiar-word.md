# «Знакомое слово!» — мост между словарём и уроками

**Задача:** ROADMAP.md → задача #35
**Статус:** ready

---

## Обучающий элемент

Слова, которые пользователь переводил в боте, появляются в теории уроков со специальной меткой.
Это создаёт сильнейший мнемонический якорь: личный контекст + учебный контекст = двойное кодирование.

**Паттерн «узнавания»:**
- Пользователь перевёл «სახლი» в боте → добавилось в словарь.
- Спустя неделю открывает урок «Падежи» → видит пример «სახლი — дом» с золотой звездой ★.
- Тап: «Ты добавил это слово 12 апреля» — момент удивления и гордости.

**Reveal-момент:** не одномоментный, а накопительный. Чем больше слов в словаре — тем чаще
пользователь видит ★ в уроках, тем сильнее ощущение «я уже знаю это!» Бесплатное мотивационное
подкрепление без геймификации.

**Образовательный эффект:** пользователь понимает, что бот-переводчик и уроки — одна система.
«Слова, которые я ищу в чате, я потом встречаю в уроках». Это меняет поведение: больше переводов
в боте, больше слов в словаре → больше ★ в уроках → петля мотивации замкнута.

## Данные

### Источник

Используется существующий endpoint `/api/miniapp/vocabulary` (GET).
Новый endpoint не нужен — все нужные поля уже есть в ответе:

```typescript
// Нужно из ответа /api/miniapp/vocabulary:
interface VocabularyWordInfo {
  word: string          // Грузинское слово (из VocabularyEntry.Word)
  dateAddedUtc: string  // ISO 8601 дата добавления
}
```

### Кэширование

Слова загружаются **один раз за сессию** и хранятся в `sessionStorage` под ключом `bombora_userwords`.
Структура: `Map<string, string>` — `{ word → dateAddedUtc }`. Слова нормализуются к нижнему
регистру для сравнения.

Логика:
1. При открытии первого `LessonTheory` — проверить `sessionStorage`.
2. Если данных нет — сделать fetch, сохранить результат.
3. Если слов 0 или fetch упал — сохранить пустой объект, больше не запрашивать до следующей сессии.
4. Последующие `LessonTheory` читают только из `sessionStorage`, без fetch.

### Матчинг слов

Сравниваем слова из теории с сохранёнными в словаре.

Приоритет — блоки типа `example` (поле `ge`): это грузинский текст в структурированном виде.
Сопоставление — **поиск подстроки** без учёта регистра:

```typescript
function matchVocab(georgianText: string, vocabWords: Map<string, string>): string | null {
  const lower = georgianText.toLowerCase()
  for (const [word] of vocabWords) {
    if (lower.includes(word.toLowerCase())) return word
  }
  return null
}
```

Если найдено совпадение — блок помечается.

**В будущем (V2):** аналогичная логика для `paragraph`-блоков со встроенными грузинскими словами
(можно выделять тегом `<geo>` в тексте на стороне бэкенда). Сейчас не в скоупе.

## Новые компоненты

### FamiliarWordBadge

```typescript
interface FamiliarWordBadgeProps {
  word: string
  dateAddedUtc: string
  onOpenVocabulary: () => void
}
```

- Рендерит золотой бейдж `★ знакомое`
- Управляет локальным `useState(false)` для открытия/закрытия FamiliarWordTooltip
- Рендерит `FamiliarWordTooltip` через `createPortal` (чтобы не обрезался родительским overflow)

### useUserWords (хук)

```typescript
function useUserWords(): Map<string, string>
// Возвращает Map<georgianWordLower, dateAddedUtc>
// Пустой Map если: пользователь не аутентифицирован / слов нет / fetch упал
```

Реализация:
1. Читает `sessionStorage.getItem('bombora_userwords')`.
2. Если есть — парсит и возвращает.
3. Если нет — делает `fetch('/api/miniapp/vocabulary')`, берёт `items` + `starterItems`,
   формирует объект `{ word.toLowerCase(): dateAddedUtc }`, сохраняет в `sessionStorage`.
4. Возвращает Map из сохранённых данных.
5. При ошибке — возвращает пустой Map, не ломает UI.

**Только для блоков `example`** (тип `TheoryBlockDto.type === 'example'`).

Изменения в `TheoryBlock` / `LessonTheory`:

```typescript
// В LessonTheory или TheoryBlock: получаем Map слов из хука
const userWords = useUserWords()

// При рендере блока example:

Если `matchedWord !== null` — рендерить `FamiliarWordBadge` в правом верхнем углу блока.

**Изменения в разметке example-блока:**
```
div.jewel-tile  [position: relative]

*Если слов 0 или хук вернул пустой Map — `FamiliarWordBadge` не рендерится нигде. Нет изменений в UI.*

## Адаптивность (375px)

- Бейдж `★ знакомое`: размер 9px текста, padding px-2 py-1 — помещается в правый угол даже
  на узкой плашке. Минимальная tap-зона обеспечена min-w-[44px].
- Tooltip (bottom sheet): ширина 100%, max-width совпадает с контейнером приложения (480px).
  Грузинское слово 28px — не переполняется даже при длинных словах (max 15 символов в грузинском).
- Кнопка «Открыть словарь» — full-width, высота 48px, tap target ок.
- Отступ снизу: `paddingBottom = 'calc(var(--safe-b) + 24px)'` — работает с Telegram safe area.

## Edge cases

| Кейс | Поведение |
|------|-----------|
| Слов в словаре 0 | Хук возвращает пустой Map. Ни одного ★ нет. Fetch всё равно делается однажды. |
| fetch упал (offline / auth) | Пустой Map, фича скрыта. Следующая попытка — при следующей сессии. |
| Несколько слов из словаря в одном блоке | Показывается ★ для первого совпадения. Tooltip показывает это первое слово. |
| Очень длинное грузинское слово | Tooltip: `word-break: break-word`, блок не переполняется. |
| Пользователь добавил слово в боте и сразу открыл урок | Кэш sessionStorage уже заполнен без нового слова. Матч появится только в следующей сессии. Это ок — лишний запрос не нужен. |
| Пользователь тапает на ★ в одном блоке, потом в другом | Первый tooltip закрывается (один activeTooltip за раз через локальный useState в каждом Badge). |
| Слово в блоке встречается как подстрока более длинного слова (e.g. «კარ» в «კარგი») | V1: принимаем — пользователю приятно узнавание; V2 можно добавить word-boundary через Unicode. |

## Доступность

- Бейдж `★ знакомое` — `role="button"` + `aria-label="Знакомое слово — открыть подробности"`.
- Tooltip — `role="dialog"` + `aria-modal="true"`, фокус перемещается внутрь при открытии.
- Кнопка закрытия ✕ — `aria-label="Закрыть"`.
- Цвета: золотой `#F5B820` на чёрном `#15100A` — контраст WCAG AA для крупного текста (9px uppercase ≈ bold ≥ 14pt).
