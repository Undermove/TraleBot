# Паттерн дня

**Задача:** ROADMAP.md → задача #37
**Статус:** ready

---

## Обучающий элемент

Каждое утро в 09:00 МСК бот присылает один грамматический паттерн — суффикс, послелог или
спрягательный шаблон — с тремя живыми примерами.

Механика: **пассивный ежедневный контакт** с грамматической структурой. Пользователь видит
паттерн утром, не открывая мини-апп. Когда позже встречает ту же структуру в уроке —
срабатывает **эффект узнавания** («это же то самое -ში!»). Долгосрочная цель: 30+ паттернов
за месяц покрывают 70% грамматики A1→A2.

**Reveal-момент:** пользователь видит послелог -ში в квизе VoM или уроке «Выживание» и
понимает: «я это уже знаю — бот показывал». Узнавание без зубрёжки.

**Обучающий элемент в боте:** заголовок «სიტყვა-ი დღი-ს-ა» (слово дня — в груз. родит. падеже)
повторяется каждый день → пользователь запоминает форму родительного неосознанно.

## Telegram-сообщение: формат

Бот поддерживает MarkdownV2. Сообщение компактное — читается за 10 секунд.

### Структура сообщения

```
📐 *ПАТТЕРН ДНЯ*

*\-ში* — *в/внутри*
_суффикс места \(локатив\)_

• სახლ*ში* — дома, в доме
• მანქანა*ში* — в машине
• ქალაქ*ში* — в городе

💡 _Приклеивается к существительному без пробела\._
```

**Правила форматирования:**
- Первая строка: `📐 *ПАТТЕРН ДНЯ*` — константный заголовок (иконка постоянна, не вращается)
- Паттерн: `*-суффикс*` жирным + `— *перевод*`
- Тип паттерна: курсив `_послелог_, _суффикс падежа_, _суффикс времени_`
- Три примера: bullet `•`, грузинское слово с выделенным паттерном жирным + перевод через `—`
- Финальная заметка: `💡 _...\_` — одно предложение, кратко (опционально, для части паттернов)
- InlineKeyboard: одна кнопка `[Практиковать 📖]`

### Варианты структуры для разных типов паттернов

**Суффикс-постпозиция (пространственный):**
```
📐 *ПАТТЕРН ДНЯ*

*\-ზე* — *на/о*
_суффикс места и темы_

• მაგიდა*ზე* — на столе
• სახლ*ზე* — о доме
• ენა*ზე* — о языке

💡 _Один суффикс — два значения: место и тема разговора\._
```

**Форма спряжения (глагольный шаблон):**
```
📐 *ПАТТЕРН ДНЯ*

*ვ\.\.\.\-ებ* — *я \.\.\. \(наст\. время\)*
_1 л\. ед\. ч\. — суффиксный шаблон_

• *ვ*საუბრ*ებ* — я говорю
• *ვ*მუშა*ობ* — я работаю
• *ვ*სწავლ*ობ* — я учусь

💡 _ვ\- в начале = «я»\. Всегда\._
```

**Падежная форма:**
```
📐 *ПАТТЕРН ДНЯ*

*\-ს* — *дательный падеж*
_кому / чему / для кого_

• კაც*ს* — мужчине, человеку
• ბავშვ*ს* — ребёнку
• მეგობარ*ს* — другу

💡 _Инверсные глаголы \(მინდა, მიყვარს\) управляют дательным\._
```

## InlineKeyboard

```
[ Практиковать 📖 ]
```

- Одна кнопка типа `url`, открывает мини-апп через Telegram deep link
- URL-шаблон: `https://t.me/tralebot/miniapp?startapp=module_{moduleId}`
- `moduleId` — связанный модуль для каждого паттерна (см. каталог ниже)
- Если паттерн не привязан к конкретному модулю — открывается Dashboard (без `startapp`)

**Примечание:** параметр `startapp` передаётся в `initData` → фронтенд парсит и
переходит к нужному модулю. Реализация deep link route — задача Developer-а; дизайн
предполагает наличие этого механизма (аналогичен существующему URL-роутингу в Mini App).

## Бэкенд: scheduled job

**Файл для реализации:** `src/Infrastructure/Telegram/ScheduledJobs/PatternOfDayJob.cs`

```
Каждый день в 09:00 МСК:
1. Получить список активных пользователей (MiniAppEnabled = true у бота)
2. Вычислить индекс паттерна: dayOfYear % 30
3. Получить паттерн[index] из статичного конфига
4. Сформировать Markdown-сообщение по шаблону
5. Для каждого пользователя: SendMessage(chatId, text, parseMode=MarkdownV2,
   replyMarkup=InlineKeyboard([Практиковать 📖, url=deepLink]))
```

**Условие отправки:**
- Пользователь должен иметь `ReceiveDailyPatterns = true` (новый opt-in флаг, дефолт `true`)
- Или отправлять всем, кто получает другие уведомления бота (согласно текущей логике)
- **Не отправлять** если `MiniAppEnabled = false` у конфигурации сервера

**Конфигурация расписания:** `appsettings.json`
```json
"PatternOfDay": {
  "Enabled": true,
  "SendAtHourMsk": 9,
  "SendAtMinuteMsk": 0
}
```

## Экраны мини-аппа (минимальные изменения)

Новый экран **не нужен**. Кнопка «Практиковать» открывает существующий модуль через deep link.

### Обработка startapp-параметра (уже частично существует)

```typescript
// В App.tsx или router-слое:
const startapp = getTelegramInitData()?.start_param   // "module_survival"
if (startapp?.startsWith('module_')) {
  const moduleId = startapp.replace('module_', '')
  navigateTo(`/module/${moduleId}`)
}
```

Если `moduleId` невалиден или параметра нет → Dashboard.

## Состояния и edge cases

| Сценарий | Поведение |
|----------|-----------|
| Пользователь не подписан на бот | Не получает сообщение |
| `MiniAppEnabled=false` на сервере | Job не запускается вовсе |
| Кнопка нажата на устройстве без Telegram | Fallback — обычный URL бота |
| Паттерн #30, следующий день = #1 | Ротация: `dayOfYear % 30` — циклична |
| Пользователь хочет отписаться | `/settings` в боте → «Отключить паттерн дня» |

**Правила copy:**
- Без «молодец», «давай», «вперёд» — нейтрально и информационно
- Заголовок паттерна всегда кириллицей: «в/внутри», «дательный падеж» — не транслитерация
- Грузинский — в примерах, не в инструкции
- Подсказка `💡` — максимум одно предложение, без восклицательных знаков

## Адаптивность / платформа

Сообщение Telegram — не мини-апп, адаптивность управляется клиентом Telegram.

**Ограничения MarkdownV2:**
- Спецсимволы `( ) . ! - |` и другие должны быть экранированы `\`
- Жирный: `*текст*`, курсив: `_текст_`, моноширинный: `` `текст` ``
- Нет кастомных цветов, шрифтов — только форматирование Telegram

**Tap target кнопки:** InlineKeyboard-кнопка Telegram всегда ≥ 44px высоту — соответствует норме.
