## Общие правила

1. Все грузинские токены — только мхедрули (U+10D0–U+10FF). Никаких кириллических/латинских/армянских/CJK символов.
2. Каждый токен из `correctOrder` ОБЯЗАН присутствовать в `chipPool`.
3. `chipPool` = правильные токены + 2–3 дистрактора. Дистракторы должны быть лингвистически правдоподобной альтернативой (неправильный падеж/постпозиция/глагольная форма) — не случайные слова.
4. `hint` ≤ 36 символов на русском, фокусируется на самой типичной ошибке этой пары.
5. Прогрессия per modulе: L1 = один пустой слот (распознавание), L2 = два пустых слота (выбор формы + согласование), L3 = три+ пустых слота / половина предложения (продуктивная сборка).

## 1. Cases — Урок 10 «Эргативный и дательный падежи»

**Pedagogical focus:** Эргатив (-მა) у subject'а переходного глагола в аористе; датив (-ს) у direct object'а Класса 1 в настоящем времени. Принципиально различает «делал» (ImpfPres NOM-DAT) от «сделал» (Aor ERG-NOM).

### L1 (5 пар, один пустой слот)

| Russian | Georgian canonical | Empty slot | Hint |
|---|---|---|---|
| Мужчина прочитал книгу | **[კაცმა]** წაიკითხა წიგნი | კაცმა (ERG) | Подлежащее в эргативе |
| Девушка открыла дверь | **[გოგომ]** გააღო კარი | გოგომ (ERG) | -მა маркирует деятеля |
| Я ем яблоко | მე ვჭამ **[ვაშლს]** | ვაშლს (DAT) | Объект в дативе |
| Учитель пишет письмо | მასწავლებელი წერს **[წერილს]** | წერილს (DAT) | -ს у прямого объекта |
| Кот поймал мышку | **[კატამ]** დაიჭირა თაგვი | კატამ (ERG) | В аористе подлеж. в эрг. |

**Distractors:** другие падежные формы того же существительного (NOM `კაცი`, GEN `კაცის`, INSTR `კაცით`); неправильный аорист (`კაცი წაიკითხა` без -მა); пустые/случайные чипы того же семантического поля.

### L2 (3 пары, два пустых слота)

| Russian | Georgian canonical | Slots |
|---|---|---|
| Студент написал статью | **[სტუდენტმა]** დაწერა **[სტატია]** | ERG-subject + NOM-object |
| Мама готовит ужин | **[დედა]** ამზადებს **[ვახშამს]** | NOM-subject + DAT-object |
| Бомбора видит собаку | **[ბომბორა]** ხედავს **[ძაღლს]** | NOM-subject + DAT-object |

**Note for native-reviewer:** разница NOM-DAT (present) vs ERG-NOM (aorist) — ключевой контраст. Дистракторы должны включать перепутанные формы (`სტუდენტი` без -მა; `ვახშამი` без -ს).

### L3 (3 пары, 3+ пустых слотов)

| Russian | Georgian canonical | Tokens |
|---|---|---|
| Дед рассказал внуку историю | ბაბუამ უამბო შვილიშვილს ისტორია | ERG + verb + DAT-recipient + NOM-object |
| Вчера учитель проверил работы | გუშინ მასწავლებელმა შეამოწმა ნამუშევრები | ADV + ERG + verb + NOM-pl |
| Друг подарил цветы маме | მეგობარმა აჩუქა ყვავილები დედას | ERG + verb + NOM-object + DAT-recipient |

## 2. Present Tense — Урок 7 «Собери предложение в SOV»

**Pedagogical focus:** Канонический грузинский порядок Subject-Object-Verb с глаголами Класса 1 (переходные: ვკითხულობ, ვწერ, ვხედავ) и Класса 2 (непереходные: მივდივარ, ვძინავ, ვცხოვრობ). Глагол всегда в конце.

### L1 (5 пар, один пустой слот)

| Russian | Georgian canonical | Empty slot | Hint |
|---|---|---|---|
| Я читаю книгу | მე **[ვკითხულობ]** წიგნს | ვკითხულობ (Cl1, 1sg) | -ვ- у 1 лица |
| Ты пишешь письмо | შენ **[წერ]** წერილს | წერ (Cl1, 2sg) | 2 лицо без префикса |
| Бомбора идёт домой | ბომბორა **[მიდის]** სახლში | მიდის (Cl2, 3sg) | Глагол в конце |
| Мы видим Тбилиси | ჩვენ **[ვხედავთ]** თბილისს | ვხედავთ (Cl1, 1pl) | -თ маркирует мн.ч. |
| Кошка спит на диване | კატა **[სძინავს]** დივანზე | სძინავს (Cl2/Cl3, 3sg) | Глагол всегда последний |

**Distractors:** глаголы в неправильном лице (`ვკითხულობ` для 2sg); порядок SVO (`მე წიგნს ვკითხულობ` — формально тоже встречается, но не SOV эталон).

### L2 (3 пары, два пустых слота)

| Russian | Georgian canonical | Slots |
|---|---|---|
| Студент пишет тест | სტუდენტი **[წერს]** **[ტესტს]** | verb + DAT-object |
| Бомбора идёт в школу | ბომბორა **[მიდის]** **[სკოლაში]** | verb + locative |
| Я делаю чай | მე **[ვაკეთებ]** **[ჩაის]** | verb-1sg + DAT-object |

### L3 (3 пары, 3+ слотов)

| Russian | Georgian canonical | Tokens |
|---|---|---|
| Утром мама пьёт кофе | დილით დედა სვამს ყავას | ADV + S + V + O |
| Брат пишет учителю письмо | ძმა წერს მასწავლებელს წერილს | S + V + IO + DO |
| Кошки видят собаку в парке | კატები ხედავენ ძაღლს პარკში | S-pl + V-3pl + DAT + LOC |

## 3. Cafe — Урок 7 «Собери заказ»

**Pedagogical focus:** Шаблоны «მინდა X» (хочу X) и «ერთი X თუ შეიძლება» (один X пожалуйста). Лексика напитков и базовых блюд.

### L1 (5 пар, один пустой слот)

| Russian | Georgian canonical | Empty slot | Hint |
|---|---|---|---|
| Я хочу кофе | მე მინდა **[ყავა]** | ყავა | Объект после მინდა |
| Я хочу чай | მე მინდა **[ჩაი]** | ჩაი | NOM, без -ს |
| Один хачапури пожалуйста | ერთი **[ხაჭაპური]** თუ შეიძლება | ხაჭაპური | Числительное + сущ. |
| Я хочу воду | მე მინდა **[წყალი]** | წყალი | Грузинское «вода» |
| Меню пожалуйста | **[მენიუ]** თუ შეიძლება | მენიუ | Запрос меню |

### L2 (3 пары, два пустых слота)

| Russian | Georgian canonical | Slots |
|---|---|---|
| Один кофе и хинкали | **[ერთი]** ყავა და **[ხინკალი]** | quantifier + dish |
| Я хочу чай с лимоном | მე **[მინდა]** ჩაი **[ლიმონით]** | verb + INSTR-modifier |
| Бомбора заказывает хачапури | ბომბორა **[უკვეთავს]** **[ხაჭაპურს]** | Cl1-verb + DAT-object |

### L3 (3 пары, 3+ слотов)

| Russian | Georgian canonical | Tokens |
|---|---|---|
| Принесите счёт пожалуйста | ანგარიში მოიტანეთ თუ შეიძლება | object + imp + politeness |
| Я хочу хачапури с сыром | მე მინდა ხაჭაპური ყველით | S + V + O + INSTR |

## 4. Shopping — Урок 7 «Сколько стоит?»

## Реализация (для dev-агентов)

Каждый из пяти JSON-файлов (`questions{N}.json` в соответствующих модулях) должен:
- Содержать ≥ 5 вопросов суммарно (L1 + L2 + L3)
- Каждый вопрос имеет `questionType: "sentence-builder"` и `level: 1|2|3` соответственно
- `targetSentence.ru` = русская фраза-эталон из таблицы выше
- `correctOrder` = массив грузинских токенов в правильном порядке (буквально как в колонке Georgian canonical)
- `presetPositions` = массив `{position, token}` для preset-чипов (в L1 их 2, в L2 — 1, в L3 — 0). Для L1 типичный preset = подлежащее + глагол; в L2 — только подлежащее.
- `chipPool` = `correctOrder` + 2–3 дистрактора из «Distractors» секции
- `hints` = объект `{<position>: "русский хинт ≤36 символов"}` для slot'ов с типичной ошибкой

Пример скелета (Cases L10, L1 q01):

```json
{
  "id": "cases10-l1-q01",
  "questionType": "sentence-builder",
  "level": 1,
  "lemma": "კაცმა",
  "question": "Собери предложение: Мужчина прочитал книгу",
  "targetSentence": { "ru": "Мужчина прочитал книгу" },
  "correctOrder": ["კაცმა", "წაიკითხა", "წიგნი"],
  "presetPositions": [
    { "position": 1, "token": "წაიკითხა" },
    { "position": 2, "token": "წიგნი" }
  ],
  "chipPool": ["კაცმა", "კაცი", "კაცის", "კაცით", "წაიკითხა", "წიგნი"],
  "hints": { "0": "Подлежащее в эргативе" },
  "answer_index": 0,
  "explanation": "Эргатив (-მა) у подлежащего переходного глагола в аористе.",
  "options": []
}
```

`MaxLessons` в `ModuleRegistry.cs` поднять на единицу для каждого модуля (`cases` 9→10, `present-tense` 6→7, `cafe` 6→7, `shopping` 6→7, `taxi` 6→7). В `MiniAppContentProvider.cs` — добавить `Lesson(N, "<title>", ...)` в соответствующий `Build*Module()` с теоретическим блоком (одна-две памятки + List с ключевыми формами).

## Открытые вопросы для native-reviewer'а

1. **Cases L1 q5** «Кот поймал мышку» — `კატამ დაიჭირა თაგვი`. Уверены ли, что `დაიჭირა` (Class 1, aorist) — естественный для речи? Может быть `დაიჭირა` слишком книжно, и стоит `კატამ შეიჭამა თაგვი` (съел) для нейтрального регистра.
2. **Cafe L2 q3** «Бомбора заказывает хачапури» — `უკვეთავს` редко используется в кафе-речи. Реально клиент говорит `მე მინდა` или `ერთი ხაჭაპური`. Возможно стоит заменить эту пару.
3. **Shopping L3 q3** — конструкция `ჩემთვის ძვირია` (для меня дорого) — нативная или калька с русского? Альтернатива: `ეს ძალიან ძვირია`.
4. **Taxi L1 q5** «Поверните направо» — `მარჯვნივ მობრძანდით` использует вежливую форму глагола движения. Для такси чаще говорят `მარჯვნივ გადაუხვიეთ`. Проверить регистр.

После native-reviewer'а distractor-pool каждого вопроса должен быть ещё раз перечитан — для каждого дистрактора native подтверждает, что он действительно ошибочный (не образует второго грамматически верного предложения с тем же смыслом).
