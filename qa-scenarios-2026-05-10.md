## Закрытые таски

### #886 — Tap-only: одиночный тап по чипу

**Что проверяем:** при тапе на чип из пула он мгновенно вставляется в первый свободный слот без двухшагового выбора; тап по заполненному слоту возвращает чип в пул; при заполненных слотах лишний тап вызывает анимацию встряхивания.

Навигация для #886 и #887 одинакова — см. блок «Как добраться до упражнения» ниже.

**Как добраться до упражнения:**

1. Открой мини-апп в @traletest_bot.
2. На главном экране найди плитку «Послелоги» 📍 и тапни на неё.
3. В списке уроков тапни на Урок 7 (он последний в модуле).
4. Нажми «Начать урок» и пройди вопросы, пока не появится упражнение «Собери предложение».

Первые вопросы в уроке — L1/L2/L3 (множественный выбор, вписывание слова). Конструктор предложений появляется ближе к концу урока. Можно ускориться: пройди несколько вопросов, пока не увидишь экран с серыми слотами-ячейками вверху и чипами-словами внизу.

**Шаги:**

1. Когда появится экран с «Собери предложение» — убедись, что чипы в нижней зоне («სიტყვები») ещё не выбраны, слоты пусты.
2. Тапни один раз на любой чип. Ожидаемо: чип сразу занимает первый пустой слот (крайний левый). Промежуточного состояния «чип выделен» не должно быть — он должен переместиться мгновенно.
3. Тапни ещё на один чип. Ожидаемо: он попадает в следующий свободный слот по порядку.
4. Тапни на уже заполненный слот (тот, куда только что вставил чип). Ожидаемо: чип возвращается в нижнюю зону, слот становится пустым.
5. Заполни все слоты до конца, пока кнопка «Проверить» не станет активной (тёмно-синей). Тапни «Проверить».
6. Теперь, когда все слоты заняты, тапни на оставшийся чип-дистрактор в нижней зоне. Ожидаемо: чип на мгновение (~150 мс) «встряхивается» и возвращается на место; слоты не меняются; кнопка «Проверить» остаётся активной.

**Ожидаемое поведение к концу:** нет двухшаговой логики «выбрал → тапнул по слоту»; каждый тап по чипу сразу меняет состояние; shake-анимация хорошо видна при тапе на лишний чип.

**Если что-то не так:** открой DevTools → Console. Ошибок при тапах быть не должно. Если чип остаётся «висеть» выделенным после тапа — старая логика `chip-selected` не удалена.

### #887 — No-scroll layout: раскладка без горизонтального скролла

**Что проверяем:** слоты переносятся на следующую строку при нехватке места, чипы в нижней зоне отображаются в 3 колонки, при 7+ слотах шрифт вопроса становится меньше — на 375 px нет горизонтальной полосы прокрутки.

Навигация — та же, что в #886: **«Послелоги» 📍 → Урок 7 → «Собери предложение»**.

**Шаги:**

1. Открой любое упражнение «Собери предложение» (подойдёт любой модуль с уроком 7).
2. Посмотри на зону с чипами («სიტყვები»). Ожидаемо: чипы расположены в сетке из 3 колонок, а не в один горизонтальный ряд.
3. Посмотри на зону слотов. Если предложение длинное (5–6 слотов и более), слоты должны переноситься на следующую строку при нехватке ширины — горизонтальной полосы прокрутки быть не должно.
4. Проверь, что страница не «убегает» вправо: свайпни влево — экран должен быть неподвижен (нет overflow).
5. Для проверки уменьшения шрифта: найди любой вопрос с коротким предложением (3–4 слота) — текст вопроса будет крупным (≈22 px). Затем найди вопрос с 6 слотами — текст вопроса будет чуть меньше (≈18 px). Вопрос с 7 и более слотами проверяется только автотестами Playwright (в JSON-контенте максимум 6 слотов).

**Ожидаемое поведение к концу:** ни в одном упражнении SentenceBuilder не появляется горизонтальная прокрутка; чипы всегда идут в 3 колонки.

**Если что-то не так:** открой DevTools → Computed styles на контейнере чипов (`[data-testid="chip-pool"]`) и проверь `grid-template-columns` — должно быть три одинаковых значения. На контейнере слотов (`[data-testid="sentence-slot-row"]`) должен быть `flex-wrap: wrap`.

### #888 — Контент: вопросы L4 и L5 в 6 модулях

**Что проверяем:** в каждом из 6 модулей появились новые вопросы уровня L4 (глагол preset, остальные слоты пусты) и L5 (все слоты пусты, ученик строит предложение с нуля).

Быстрая проверка через браузер (рекомендуется — не нужно проходить уроки целиком):

Открой в браузере следующие URL по очереди и убедись, что ответ содержит объекты с полями `"level": 4` и `"level": 5`:

- Послелоги: http://localhost:1402/api/miniapp/modules/postpositions/lessons/7/questions
- Падежи: http://localhost:1402/api/miniapp/modules/cases/lessons/10/questions
- Настоящее время: http://localhost:1402/api/miniapp/modules/present-tense/lessons/7/questions
- В кафе: http://localhost:1402/api/miniapp/modules/cafe/lessons/7/questions
- Магазин: http://localhost:1402/api/miniapp/modules/shopping/lessons/7/questions
- Такси и город: http://localhost:1402/api/miniapp/modules/taxi/lessons/7/questions

В каждом ответе должно быть минимум 2 объекта с `"level": 4` и минимум 1 объект с `"level": 5`.

**Ручная проверка UI (необязательно, но желательно для одного модуля):**

1. Открой мини-апп → «В кафе» → Урок 7 → «Начать урок».
2. Пройди вопросы до появления «Собери предложение» с L4-уровнем. Узнать его можно так: в упражнении один слот будет уже заполнен (глагол «მინდა» — «хочу»), остальные слоты пусты.
3. Убедись, что заполненный слот нельзя убрать тапом (он preset-серый).
4. Заполни оставшиеся слоты из пула: «ცხელი» («горячий»), «ჩაი» («чай»), «ლიმონით» («с лимоном»). Перевод: «Я хочу горячий чай с лимоном».
5. Нажми «Проверить» — должен появиться зелёный баннер «Отличная работа!».
6. Продолжи урок до вопроса L5 — в нём все слоты пусты (ни один не заполнен заранее). Попробуй составить «Я хочу хачапури с сыром»: «მე», «მინდა», «ხაჭაპური», «ყველით».

**Ожидаемое поведение к концу:** в каждом из 6 модулей есть как минимум 3 вопроса L4 и 1 вопрос L5; L4-вопросы содержат ровно один preset-слот (глагол), L5-вопросы — полностью пустые слоты.

**Если что-то не так:** открой Network → Response для запроса `/questions` и проверь, что в массиве есть `"level": 4` и `"level": 5`. Если их нет — билд wwwroot мог остаться старым, перезапусти `dotnet run`.

### #889 — Бэкенд: прогрессионный гейт L4/L5

**Что проверяем:** добавлена миграция базы данных с новой колонкой `SentenceBuilderProgressJson`, и реализована доменная логика прогрессионного гейта (unit-тесты покрывают все 4 случая: L4 заблокирован при несданном L3, L4 разблокирован при освоении L3, и аналогично для L5).

**Важное примечание:** логика гейта реализована как чистый сервис (`SentenceBuilderProgressionGate.FilterByProgression`) и покрыта unit-тестами, однако в текущем прогоне она ещё **не подключена** к API-эндпоинту вопросов — вопросы L4 и L5 появляются у всех пользователей без ограничения по прогрессу. Это фундамент для следующего шага (подключение гейта в контроллере).

**Как проверить (backend-только):**

Шаг 1 — миграция применена. Открой в браузере или через curl:

`http://localhost:1402/api/miniapp/modules/cases/lessons/10/questions`

Если API отвечает 200 и возвращает JSON — это означает, что EF Core подключился к БД успешно и новая колонка `SentenceBuilderProgressJson` присутствует в таблице `MiniAppUserProgresses`. Если база не обновилась, EF бросил бы исключение при запуске.

Шаг 2 — убедись, что в ответе присутствуют `"level": 4` и `"level": 5` (это подтверждает вместе с задачей #888, что контент загружается). Пример curl-запроса для проверки:

`curl -s http://localhost:1402/api/miniapp/modules/cases/lessons/10/questions | python3 -m json.tool | grep '"level"'`

В выводе должны быть строки `"level": 4` и `"level": 5`.

Шаг 3 — проверь unit-тесты гейта. В репозитории запусти:

`dotnet test tests/Application.UnitTests/ --filter "SentenceBuilderProgressionGate" --verbosity normal`

Ожидаемо: все тесты зелёные (покрыты сценарии L4-blocked, L4-unlocked, L5-blocked, L5-unlocked, L4-still-blocked-after-one-correct, L5-blocked-when-only-L3-mastered).

**Ожидаемое поведение к концу:** API работает без ошибок, в ответе есть L4/L5-вопросы, unit-тесты проходят. Видимый пользовательский эффект гейтирования (L4/L5 не показываются, пока не освоен L3) появится в следующем прогоне.

**Если что-то не так:** если API отдаёт 500 — проверь логи `dotnet run` на ошибки EF-миграции; возможно, нужно выполнить `dotnet ef database update --project src/Persistence/Persistence.csproj --startup-project src/Trale/Trale.csproj`.

4. Убедись, что кнопка «Проверить» становится активной после заполнения всех слотов.
5. Нажми «Проверить» и убедись, что правильные слоты подсвечиваются зелёным, неправильные — красным.
