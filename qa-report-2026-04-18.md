### Найденные потенциальные проблемы

#### ⚠️ P3 — Глобальный `toastTimer` в `ComingSoonTile`

**Описание:** переменная `toastTimer` объявлена на уровне модуля, то есть shared между всеми экземплярами компонента. Если пользователь быстро нажимает на два разных «Coming Soon» тайла:
1. Тайл A: показывает toast, устанавливает `toastTimer`
2. Тайл B: очищает `toastTimer` (таймер A отменяется), показывает свой toast
3. Toast A остаётся на экране бесконечно, пока B не исчезнет

**Влияние:** оба `fixed`-toast могут отображаться одновременно, UI выглядит замусоренным. Маловероятно на практике (нужно быстро тапать два тайла), но возможно.

**Рекомендация:** перенести `toastTimer` внутрь компонента через `useRef`, чтобы таймер был per-instance. Небольшой фикс, но не блокер для запуска.

```tsx
// вместо module-level let toastTimer
const toastTimerRef = useRef<ReturnType<typeof setTimeout> | null>(null)
```
