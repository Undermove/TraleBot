# Стратегия — Бомбора (TraleBot mini-app)

> Короткий документ про «зачем мы сейчас что-то делаем». Обновляется по мере смены фаз. Продакт-агент читает это каждую сессию и приоритизирует backlog под текущую фазу. Все остальные агенты — справочно.

## Текущая фаза: **Pre-launch**

Цель ближайшей фазы — **запустить мини-апп и маркетинг**. Не расширять скоуп, не копить идеи про запас, а закрыть список ниже и выйти в стор.

## Launch-чеклист

Задачи здесь имеют приоритет выше любых новых идей. Продакт-агент обязан сверяться с ним перед тем, как предлагать что-то новое.

### 1. Контент проверен методистом

- [ ] Алфавит — прошёл проверку (уже закрыто)
- [ ] Числительные — прошли проверку (уже закрыто)
- [ ] Глаголы движения — модуль вычитан, порядок уроков корректен, формулировки без двусмысленностей
- [ ] Мой словарь — fallback-набор для пустого словаря подобран корректно
- [ ] Все quiz-вопросы имеют правильные ответы и понятные формулировки

### 2. Интерфейс чистый, ничего лишнего

- [ ] Нет дублирующих кнопок (пример: «×» когда уже есть BackButton от Telegram и стрелка ←)
- [ ] Везде одна навигационная парадигма (Telegram BackButton + стрелка ←, никаких «крестиков»)
- [ ] Экраны без перегруженности: один первичный CTA, один вторичный
- [ ] Minankari-дизайн применён единообразно (палитра, типографика, один акцент на смысл)
- [ ] Нет визуальных багов и обрезанных текстов на популярных экранных размерах

### 3. Механика удержания — кормление Бомборы

Тамагочи-механика маскота как единственный механизм удержания (вместо сердечек/страйков):

- [ ] Бомбора «голодает» между сессиями, просит покормить
- [ ] «Корм» = выполненные уроки/квизы
- [ ] Разные настроения маскота в зависимости от сытости (moods уже есть: happy/cheer/sleep/think — использовать)
- [ ] Без push-шейминга, без «ты пропустил день» — только положительный фидбек
- [ ] Понятная связь: «покормил → маскот доволен → ты запомнил слова»

### 4. Монетизация

- [ ] Pay-once через Telegram Stars (XTR) — не внешний провайдер
- [ ] Понятный checkout flow, одна кнопка «Открыть полный доступ»
- [ ] Контент до покупки: первый модуль открыт, остальное с замком
- [ ] Восстановление покупки при переустановке

### 5. Запуск в прод

- [ ] `BotConfiguration.MiniAppEnabled` → `true` в `deploy/tralebot.yml`
- [ ] `SetChatMenuButton` тестирована — все пользователи получат кнопку «🐶 Бомбора»
- [ ] Landing page `tralebot.com/` направляет на бот, метрики привязаны
- [ ] Бот анонс-сообщение существующим пользователям готово
- [ ] Telegram analytics / mini-app metrics подключены

### 6. Маркетинг

- [ ] Превью-скриншоты мини-аппа (Story-формат)
- [ ] Короткое описание фичи в Telegram-анонсах на русском
- [ ] Пост-анонс на канале TraleBot
- [ ] Пост в экспат-чатах Батуми / русскоязычных Грузии
- [ ] Ссылка-шейринг из мини-аппа — зашил рефералку

## Правила приоритизации на эту фазу

1. **Launch-чеклист выше всего.** Если пункт оттуда не закрыт — новые [idea] НЕ создаём.
2. **Минимально раздуваем бэклог.** Если в ROADMAP уже 5+ [idea] — продакт НЕ придумывает новых, фокусируется на приоритизации.
3. **Каждая не-launch идея должна явно объяснять, почему она до запуска.** Иначе — в секцию «После запуска».
4. **Дизайн учит** остаётся железным правилом — любая фича должна иметь обучающий элемент.

## После запуска (deprioritized, но не забыто)

Идеи, которые продакт-агент может парковать сюда, если они не вписываются в launch:

- Расширение тем (грамматика падежи, грузинские глаголы состояния и т.п.)
- Социальные механики (соревнования, лидерборды — с учётом «без стрик-шейминга»)
- Голосовые упражнения
- Экспорт словаря вовне
- Мини-игры поверх алфавита

## Что делает продакт-агент с этим документом

1. Читает перед любым планированием.
2. Сверяет весь открытый backlog с launch-чеклистом.
3. Помечает в ROADMAP.md задачи, закрывающие чеклист-пункты, тегом `[launch]` — такие имеют приоритет.
4. Если методист поднимает вопрос не из launch — в соответствии с правилом (3) выше решает: ускорять до launch или парковать.
5. Продакт **генерирует собственные идеи** по маркетингу, позиционированию, онбордингу, retention — не ждёт методиста. Но при этом не раздувает backlog сверх нужного.
